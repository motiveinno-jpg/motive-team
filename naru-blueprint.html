나루의 철학

▾

파이프라인 퍼스트

대시보드가 아닌 파이프라인이 메인 화면. 제품이 수출 완료까지 어디에 있는지 한눈에.

칸반 보드처럼 — 등록 → 분석 → 매칭 → 거래 → 서류 → 출하

5분 안에 첫 결과

제조사가 URL 하나 입력하면 5분 안에 AI 분석 리포트 수령. 1회 무료. 바로 가치 체감.

복잡도 = 죄

Whistle: 68 테이블, 57 EF, 300+ 함수, 15,800줄. 나루: 18 테이블, 10 EF, ~80 함수, 4 파일.

매출이 먼저

각 파이프라인 단계에 자연스러운 수익 지점. 분석(크레딧) → 매칭(구독) → 서류(건당) → 결제(에스크로).

알리바바 = 핵심 채널

입점대행은 단순 부가서비스가 아닌 제조사 여정의 핵심 경로. BASIC/PLUS 운영대행까지 풀 케어.

바우처 = 현금 엔진

수출바우처/제조혁신바우처 수행기관. 정부 돈으로 제조사에게 서비스 제공. 결과보고 자동화가 핵심.

🏪 서비스 A: 알리바바 총판운영

모티브 자체 알리바바 계정에 운영수권서(총판권)를 받아 여러 제조사의 제품을 한 스토어에서 판매. 제조사는 별도 알리바바 가입 불필요.

총판운영 구조
모티브 계정 1개 = 다수 제조사
  • 모티브가 알리바바 GGS 연간비 부담
  • 제조사별 운영수권서(총판 계약) 체결
  • 모티브 스토어에 제조사 제품 등록
  • 인콰이어리/RFQ 모티브가 1차 응대
  • 바이어 연결 → 제조사에 전달
  • 거래 성사 시 수수료 정산
제조사 입장
진입 장벽 최소화
  • 알리바바 계정 필요 없음
  • GGS 연간비 부담 없음
  • 제품 정보 + 운영수권서만 제출
  • 모티브가 모든 운영 대행
  • 바이어 매칭되면 직접 거래 진행

알리바바 데이터 동기화 아키텍처

왜 크롤링인가? 알리바바닷컴은 공개 API를 제공하지 않음. 따라서 계정에 직접 로그인하여 데이터를 수집하고, 나루 대시보드에 표시하는 방식으로 운영 현황을 공유.
크롤링 항목수집 주기나루 표시
인콰이어리 (신규/응답대기/완료)6시간제조사 대시보드 + 알림
RFQ 견적 요청6시간대시보드 + 자동 응답 초안
방문자·페이지뷰 통계24시간월간 리포트
키워드 광고 성과24시간광고 대시보드
상품별 노출·클릭24시간상품 성과 카드
Trade Assurance 주문6시간파이프라인 연결
기술 스택: Playwright (헤드리스 브라우저) → Supabase Edge Function (스케줄러) → alibaba_accounts / alibaba_sync_data 테이블 → 나루 대시보드 실시간 표시
보안: 계정 자격증명 AES-256 암호화 저장. 복호화는 크롤링 EF 내부에서만.
