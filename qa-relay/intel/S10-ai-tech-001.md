### 1-4. HSK 10자리 vs 국제 HS 6자리 처리 방법

| 구분 | 설명 |
|------|------|
| **국제 HS** | WCO 관리, 6자리. 전 세계 공통 |
| **HSK (한국)** | 관세청 관리, 10자리. 앞 6자리=국제 HS, 뒤 4자리=한국 고유 |
| **데이터 소스** | 공공데이터포털에서 HSK 전체 데이터 공개 (data.go.kr, 2026년판 제공) |
| **API** | 관세청 공식 자동분류 API는 없음. UNI-PASS/CLIP에서 검색만 가능 |
| **처리 전략** | 국제 AI 서비스(Zonos 등)로 6자리 분류 → HSK 뒤 4자리는 관세청 데이터 매핑 테이블로 보완 |
| **민간 서비스** | hs-tariff.com 등 빅데이터 기반 HSK 연관검색 제공 |

### 2-3. False Positive 처리

- **업계 평균**: 전통적 키워드 매칭 시 90%+ 오탐률
- **AI 적용 후**: NLP + Entity Resolution으로 오탐 50~90% 감소
- **Descartes**: "수 퍼센트의 분수(fractions of a percent)" 주장 (2026)
- **처리 방법**: 1) 이름 과학(Name Science) 알고리즘, 2) 글로벌 이름 희귀도 모델, 3) 비라틴 문자 음차 처리 (중국어 포함), 4) 네트워크 분석 (간접 노출 탐지)

## 3. 수출서류 AI 자동 생성

### 3-1. 서류별 AI 자동 생성 가능 수준

| 서류 | 자동 생성 수준 | 법적 효력 | 한계 |
|------|-------------|----------|------|
| **Commercial Invoice** | 90%+ 자동화 가능 | 발행자 서명 필요 | 양식은 비교적 표준화. 전자서명으로 대체 가능 |
| **Packing List** | 90%+ 자동화 가능 | 법적 구속력 낮음 | 상품 데이터 입력만 정확하면 완전 자동화 가능 |
| **Certificate of Origin** | 50~70% 자동화 | 기관발급 또는 자율발급 | 국가별 양식 상이. 한국: 기관발급(세관/상공회의소) vs 자율발급. FTA별 다른 양식 |
| **Bill of Lading** | 추출/입력 자동화 가능 | 권리증권 - 법적 효력 중대 | 전자 B/L(eBL) 2025년 글로벌 11% 채택. 대부분 원본 종이 필요 |
| **Customs Declaration** | 70~90% 자동화 | 세관 시스템 직접 연동 필요 | 55+ 국가별 다른 시스템 (한국: UNI-PASS, 미국: ACE, EU: ICS2 등) |
| **Insurance Certificate** | 70% 자동화 | 보험사 발행 필수 | 자동 발행은 보험사 API 필요 |
| **Inspection Certificate** | 불가 | 검사기관 발행 | 물리적 검사 필수 |

### 3-2. 전자 B/L (eBL) 현황 (2025)

| 항목 | 내용 |
|------|------|
| **글로벌 채택률** | 2025년 중반 기준 11% (2025년 초 5.7%에서 급증) |
| **목표** | DCSA 회원사: 2027년 50%, 2030년 100% 디지털화 |
| **법적 인정** | 영국 ETDA 2023, 인도 2025년 법안, 프랑스/독일/네덜란드 입법 완료 |
| **MLETR** | 10개국 UNCITRAL MLETR 기반 법률 제정 완료 |
| **한국** | 아직 전자 B/L 법적 인정 미비 |

### 3-4. 한계점 정리

1. **법적 효력**: AI 생성 서류에 법적 구속력 부여 불가. 전자서명/기관 인증 필수
2. **인증/도장**: 원산지증명서 = 기관 발급 필요 (한국: 세관/상공회의소)
3. **국가별 양식**: 55+ 국가 각각 다른 양식/시스템. 표준화 부재
4. **B/L 특수성**: 권리증권이므로 위조/변조 리스크. eBL 채택 아직 11%
5. **검사증명**: 물리적 검사 필수 서류는 AI 자동 생성 불가

## 4. 관세 최적화 / FTA 활용 AI

### 4-1. FTA 원산지 자동 판정 기술 현황

| 항목 | 내용 |
|------|------|
| **기술 성숙도** | **초기 실용단계** - 규칙 기반 + AI 보조 수준 |
| **핵심 과제** | 원산지결정기준(ROO)이 협정별/품목별 상이. 세번변경기준(CTC), 부가가치기준(RVC) 등 복합 판정 |
| **한국 FTA Plus** | 산업부 지원 AI 원산지 판정 서비스. SME 활용률 60% → 82% 달성 |
| **88% 불만** | 한국 중소기업 88%가 원산지 판정 어려움 호소 (복잡한 규칙, 발급 절차, 전문가 부족) |

### 4-2. Zonos FTA 기능

| 항목 | 내용 |
|------|------|
| **기능** | Landed Cost 계산 시 FTA 특혜 세율 자동 반영 |
| **방식** | 상품 HS코드 + 원산지국 → 해당 FTA 특혜 관세 자동 적용 |
| **지원** | 주요 FTA 반영 (구체적 FTA 수 미공개) |
| **가격** | $2 + 관세/세금의 10% (주문당, Guaranteed Landed Cost) |
| **한계** | 원산지 "판정"이 아닌 "적용"만 자동화. ROO 충족 여부는 사용자 확인 |

### 4-3. 한국 관세청 FTA 포탈

| 항목 | 내용 |
|------|------|
| **FTA 포탈** | customs.go.kr/ftaportalkor - 협정별/산업별/지역별 활용현황 |
| **FTA-PASS** | 무료 원산지관리시스템. 원산지 판정, 증명서 작성 지원 |
| **okFTA** | KITA 운영 FTA 통합 플랫폼. HS코드 조회, C/O 발급 안내 |
| **API** | 공개된 API 없음. 웹 포탈만 제공 |
| **C/O 발급** | 기관발급: UNI-PASS 온라인 신청 → 세관/상공회의소 발급. 자율발급: 수출자 직접 |

### 4-4. FTA 활용률 현황

| 항목 | 수치 |
|------|------|
| **한국 전체** | 약 75~82% (FTA Plus 도입 후 개선) |
| **개도국 FTA** | 상대적으로 낮음 (아세안, 베트남, 콜롬비아, 중국) |
| **미활용 원인** | 복잡한 ROO, 발급 비용 > 관세 혜택, 전문가 부족 |
| **기회** | 활용률 갭 = 휘슬 AI의 핵심 타깃. 미활용 FTA 혜택 자동 식별 |

### 4-5. 원산지증명서 자동 발급 가능성

| 방식 | 자동화 가능성 | 한계 |
|------|-------------|------|
| **기관발급** | 낮음 - 세관/상공회의소 발급 필수 | 사전 작성 자동화 가능, 최종 발급은 기관 |
| **자율발급** | 높음 - 수출자/생산자 직접 작성 | 양식 자동 채움 + 전자서명으로 자동화 가능 |
| **한-미 FTA** | 자율발급 방식 | 수출자가 직접 작성/서명 → AI 자동 작성 가능 |
| **한-EU FTA** | 인증수출자 자율발급 | 인증수출자 지위 필요 |

### 4-6. 휘슬 AI 적용 제안

| 항목 | 제안 |
|------|------|
| **즉시** | FTA 혜택 자동 알림: HS코드 + 수출국 → 적용 가능 FTA + 예상 절감액 표시 |
| **3개월** | FTA 원산지 판정 가이드: 협정별 ROO 데이터베이스 구축 + AI 판정 보조 |
| **6개월** | 자율발급 C/O 자동 생성 (한-미 FTA 등 자율발급 협정 우선) |
| **구현 난이도** | 알림: 낮음 / ROO DB: 중간 / C/O 자동 생성: 높음 |
| **수익화** | "연간 FTA 절감액 시뮬레이션" = 강력한 세일즈 훅. Enterprise 플랜 핵심 |
| **우선순위** | **지금 당장** (FTA 알림) + **6개월** (판정 보조 + C/O) |

## 휘슬 적용 로드맵 요약

### Phase 1: 지금 당장 (0~1개월)
1. **US CSL API 연동** → 바이어/셀러 제재 스크리닝 기본 기능
