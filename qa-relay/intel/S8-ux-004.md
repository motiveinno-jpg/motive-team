# S8-ux-004: 영어 랜딩 — 한국어 대비 접근성 격차 심각

## 유형: UX 개선 제안 (긴급)
## 페이지: whistle-ai.com/en vs /ko

## 비교 분석

| 항목 | 한국어 (/ko) | 영어 (/en) | 상태 |
|------|-------------|-----------|------|
| nav 랜드마크 | 1 ✅ | 0 ❌ | 심각 |
| main 랜드마크 | 1 ✅ | 0 ❌ | 심각 |
| header 랜드마크 | 0 ❌ | 0 ❌ | 둘 다 누락 |
| footer 랜드마크 | 1 ✅ | 0 ❌ | 심각 |
| 햄버거 aria-label | ✅ "메뉴 열기" | ❌ 없음 | 누락 |
| 챗봇 aria-label | ✅ "실시간 상담 채팅 열기" | 확인 필요 | — |
| H1 | 1개 ✅ | 1개 ✅ | 일치 |
| lang 속성 | ko (추정) | en ✅ | OK |
| 성능 TTFB | 673ms | 328ms | EN이 더 빠름 |
| 로드 완료 | 1,173ms | 644ms | EN이 더 빠름 |

## 결론
영어 랜딩(global-landing.html)은 한국어 랜딩(whistle-landing.html)과 별도 파일이라
접근성 개선사항이 동기화되지 않음.
**글로벌 시장 대상이므로 WCAG 준수가 더 중요** (ADA 소송 리스크).
