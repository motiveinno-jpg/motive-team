# [S4-미국바이어] i18n: GeoIP 기반 언어 감지가 navigator.language 무시 | P1 | buyer-app, i18n-detect.js

## 재현 경로
1. VPN/프록시 또는 Cloudflare edge가 독일(DE)을 반환하는 환경에서 접속
2. 브라우저 언어가 `en-US`여도 `whistle_lang`이 `de`로 설정됨
3. buyer-app.html이 독일어(Anmelden/Registrieren) 또는 일본어(ログイン/新規登録)로 표시됨

## 기대
- `navigator.language`가 `en-US`이면 영어로 표시
- GeoIP 국가 감지보다 브라우저 언어 설정이 우선해야 함

## 실제
- Cloudflare GeoIP가 독일(DE)을 반환 → `whistle_lang: "de"` 설정
- buyer-app.html이 독일어 UI 표시 (일부 경우 일본어로 표시 — 매핑 버그 가능성)
- `localStorage._lang = "de"` 인데 일본어 UI가 나오는 현상도 관찰됨 (2차 매핑 버그)

## 근본 원인
1. `i18n-detect.js`가 `/app/*` 경로에서는 early return하지만, 다른 경로(/, /buyer) 방문 시 GeoIP로 `whistle_lang`을 localStorage에 설정
2. buyer-app.html (라인 425-435)은 localStorage의 `whistle_lang`을 그대로 사용
3. **GeoIP 결과 → localStorage 캐시 → buyer-app 언어** 체인으로 오염 전파
4. 로그인 후 언어 교정 로직(라인 2567-2573)이 **한국어만 처리** — 비한국 유저는 교정 안 됨:
   ```javascript
   // 이 로직은 한국 유저만 교정. US 유저가 de로 오염되면 그대로 유지
   if(!_isKorean && _userLang === 'ko') { _userLang = 'en'; }
   else if(_isKorean && _userLang !== 'ko') { _userLang = 'ko'; }
   ```

## 제안 수정
라인 2567-2573을 국가-언어 매핑으로 확장:
```javascript
var COUNTRY_LANG_MAP = {US:'en', GB:'en', AU:'en', KR:'ko', JP:'ja', DE:'de', ...};
var expectedLang = COUNTRY_LANG_MAP[_authCountry] || 'en';
if(_userLang !== expectedLang) {
  _userLang = expectedLang;
  localStorage.setItem('whistle_lang', _userLang);
}
```
