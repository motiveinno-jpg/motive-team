# [S4-미국바이어] i18n: GeoIP 기반 언어 감지가 navigator.language 무시 | P1 | buyer-app, i18n-detect.js

## 재현 경로
1. VPN/프록시 또는 Cloudflare edge가 독일(DE)을 반환하는 환경에서 접속
2. 브라우저 언어가 `en-US`여도 `whistle_lang`이 `de`로 설정됨
3. buyer-app.html이 독일어(Anmelden/Registrieren) 또는 일본어(ログイン/新規登録)로 표시됨

## 기대
- `navigator.language`가 `en-US`이면 영어로 표시
- GeoIP 국가 감지보다 브라우저 언어 설정이 우선해야 함

## 실제
- Cloudflare GeoIP가 독일(DE)을 반환 → `whistle_lang: "de"` 설정
- buyer-app.html이 독일어 UI 표시 (일부 경우 일본어로 표시 — 매핑 버그 가능성)
- `localStorage._lang = "de"` 인데 일본어 UI가 나오는 현상도 관찰됨 (2차 매핑 버그)

## 근본 원인
- `i18n-detect.js`가 `/app/*` 경로에서는 early return하지만, 다른 경로 방문 시 `whistle_lang`을 localStorage에 설정
- buyer-app.html은 이 `whistle_lang` 값을 읽어서 UI 언어 결정
- **GeoIP 결과가 브라우저 locale보다 우선되는 구조**
