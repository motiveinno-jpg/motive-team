## 영향
- 스크린리더 사용자가 앱을 전혀 사용할 수 없음
- 키보드 전용 사용자가 메뉴/모달/폼을 탐색할 수 없음
- ADA/WCAG 컴플라이언스 미충족 → 글로벌 시장 진출 시 법적 리스크

## 수정 우선순위 (단계적)

### Phase 1 — 즉시 (구조)
1. `<nav>`, `<main>`, `<aside>` 시맨틱 태그 추가
2. 모달에 `role="dialog"`, `aria-modal="true"`, `aria-labelledby` 추가
3. 토스트/알림에 `role="alert"`, `aria-live="polite"` 추가

### Phase 2 — 단기 (인터랙션)
4. 사이드 네비 메뉴 항목에 `role="menuitem"`, 활성 메뉴에 `aria-current="page"`
5. 모달 열릴 때 포커스 트랩 구현
6. ESC로 모달 닫기 (이미 있을 수 있으나 확인 필요)

### Phase 3 — 중기 (콘텐츠)
7. 아이콘 전용 버튼에 `aria-label` 추가
8. 동적 콘텐츠 업데이트에 `aria-live` 영역 설정
9. 폼 유효성 에러에 `aria-describedby` 연결

## ✅ FIXED (2026-03-30) — Phase 1 완료
- **시맨틱 랜드마크**: sidebar → `<nav role="navigation">`, topbar → `<header role="banner">`, page-content → `<main role="main">` (양쪽 앱)
- **Skip navigation**: 탭 포커스 시 "Skip to content" 링크 표시 (양쪽 앱)
- **Toast aria-live**: `role="status" aria-live="polite"` 추가 (양쪽 앱)
- **Auth error role="alert"**: `role="alert" aria-live="assertive"` 추가 (양쪽 앱)
- **Modal accessibility**: `role="dialog" aria-modal="true" aria-labelledby` + ESC 키 닫기 (양쪽 앱)
- **Hamburger menu**: `aria-label="Toggle navigation menu"` 추가
- Phase 2-3 (menuitem roles, focus trap, form aria-describedby) 추후 진행
