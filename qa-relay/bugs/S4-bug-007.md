# [S4-미국바이어] 이전 세션의 auth 토큰이 바이어 로그인을 방해 | P1 | buyer-app.html

## 재현 경로
1. 제조사 계정(role=client)으로 /app에 로그인한 적 있는 브라우저
2. localStorage에 sb-*-auth-token이 남아있음
3. /app/buyer 접속 → buyer-app.htm이 기존 auth 토큰으로 세션 복원
4. users 테이블에서 해당 유저의 role='client' 확인
5. → /app으로 리다이렉트 (라인 2558)
6. 바이어 로그인 폼에서 새 바이어 계정으로 Sign In 해도, 이전 제조사 세션 토큰이 우선 적용

## 기대
- 바이어 앱에서 로그인하면 이전 제조사 세션을 무효화하고 바이어 세션으로 교체
- 또는 Sign In 시 기존 세션을 signOut() 후 새 세션으로 교체

## 실제
- 바이어 로그인 성공 후에도 이전 제조사 토큰이 간섭
- loadUser()가 기존 세션을 먼저 체크하여 manufacturer로 판단 → /app 리다이렉트

## 근본 원인
- buyer-app.html의 loadUser()가 signInWithPassword() 성공 후 호출됨
- 하지만 Supabase auth가 이전 세션과 새 세션을 병합하거나, 이전 세션이 아직 유효한 상태
- doLogin() 이전에 기존 세션을 signOut()하는 로직 부재

## 제안 수정
doLogin() 함수 시작 부분에 기존 세션 해제:
```javascript
function doLogin(f) {
  // Clear any existing manufacturer session first
  sb.auth.signOut().then(function() {
    // ... proceed with signInWithPassword
  });
}
```
