# [S5-동남아바이어] P1: html lang="ko" 하드코딩 — SEO/접근성 문제

- 심각도: P1 (SEO + 스크린리더 장애)
- 뷰포트: 전체
- 페이지: /app/buyer

## 문제
`<html lang="ko">` 하드코딩 → 사용자 언어와 무관하게 항상 한국어로 선언됨.
- 스크린리더가 한국어 발음으로 영어 텍스트를 읽음
- 검색엔진이 페이지를 한국어로 인식
- WCAG 3.1.1 위반 (Language of Page)

## 코드 위치
buyer-app.html 최상단 `<html lang="ko">`

## 제안
`<html lang="">` 로 시작 후, _userLang 결정 시점에 동적 설정:
```javascript
document.documentElement.lang = _userLang;
```
