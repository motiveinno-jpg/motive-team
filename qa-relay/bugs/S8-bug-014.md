# S8-bug-014: 비한국어/비영어 접속 시 현지어 콘텐츠 없음 — 영어 fallback만 표시

## 심각도: P1
## 페이지: 전체 (랜딩 + 앱)
## 유형: i18n / UX

## 설명
`?lang=ja`, `?lang=de`, `?lang=vi` 등으로 접속하면:
- `<html lang="ja/de/vi">` → 올바르게 설정됨 ✅
- `localStorage('whistle_lang')` → 올바르게 저장됨 ✅
- **하지만 실제 페이지 콘텐츠는 전부 영어** ❌

### 테스트 결과

| 언어 | ?lang= | htmlLang | H1 텍스트 | 현지어 표시 |
|------|--------|----------|----------|-----------|
| 한국어 | ko | ko | 사진 한 장이면... | ✅ 한국어 |
| 영어 | en | en | Export Smarter with AI | ✅ 영어 |
| 일본어 | ja | ja | Export Made Intelligent | ❌ 영어로 표시 |
| 독일어 | de | de | Export Made Intelligent | ❌ 영어로 표시 |
| 베트남어 | vi | vi | Export Made Intelligent | ❌ 영어로 표시 |

### 문제의 영향
1. 일본, 독일, 베트남 등에서 접속 시 첫인상 이탈
2. `htmlLang="ja"`인데 영어 콘텐츠 → 스크린리더가 일본어 발음으로 영어를 읽음 (접근성 위반)
3. 13개 언어 지원 선언(supported array)과 실제 구현 사이 격차

### 현재 구현 상태
- **whistle-main.html**: 루트(/) 랜딩은 한/영 2개 언어만 실제 번역 (line 217 T.en 딕셔너리)
- **whistle-app.html**: `_T` 딕셔너리에 13개 언어 지원이 선언되어 있으나, 랜딩에는 미적용
- **whistle-landing.html** (한국어 랜딩): `L()` 함수로 한/영 전환만
- **global-landing.html** (영어 랜딩): 영어 전용

### 라우팅 구조 문제
- `/ko` → whistle-landing.html (한국어 전용)
- `/en` → global-landing.html (영어 전용)
- `/` → whistle-main.html (한/영만, 일본어 등은 영어 fallback)
- **일본어 전용 랜딩이 없음** → `/ja` 경로 미존재

## 수정 제안 (단계적)

### Phase 1 — 즉시
- `htmlLang`을 실제 콘텐츠 언어와 일치시키기 (영어 fallback이면 `lang="en"` 유지)
- 또는 "이 페이지는 현재 영어로 제공됩니다" 안내 배너 표시

### 중요: 앱은 다국어 작동함!
- `/app?lang=ja` → **일본어 로그인 화면 정상 표시** (ログイン, メールアドレス, パスワード)
- 앱(whistle-app.html)의 `_T` 딕셔너리에 13개 언어 번역이 있음
- **문제는 랜딩 페이지만** 한/영 2개 언어 전용으로 만들어져 있음

### Phase 2 — 단기
- whistle-main.html에 주요 5개 언어(ja, zh, vi, de, es) H1/CTA 번역 추가
- 앱의 `_T` 딕셔너리를 랜딩에도 공유

### Phase 3 — 중기
- 각 언어별 전용 랜딩 페이지 생성
- 또는 whistle-landing.html을 다국어 확장 (L() 함수에 언어 추가)
