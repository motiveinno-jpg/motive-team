# S8-bug-021: 메인 랜딩(/) — fade-up 애니메이션 미트리거 시 빈 페이지 표시

## 심각도: P2
## 페이지: whistle-ai.com/ (whistle-main.html)
## 유형: 접근성 + UX

## 설명
메인 랜딩 페이지(/)의 37개 콘텐츠 요소가 `fade-up` CSS 클래스로 초기 `opacity: 0`으로 설정.
Intersection Observer가 스크롤 시 트리거하여 표시하도록 설계.

### 문제 발생 조건
1. Headless 브라우저 (봇/크롤러)
2. JS 비활성화 또는 로딩 지연
3. Intersection Observer 미지원 환경 (IE11 등)
4. prefers-reduced-motion 설정된 환경에서 애니메이션 차단 시

### 영향
- 네비게이션만 보이고 본문 콘텐츠가 전혀 안 보이는 빈 페이지 표시
- SEO 크롤러가 콘텐츠를 못 읽을 수 있음 (opacity: 0은 hidden으로 해석될 수 있음)
- AI 검색엔진(ChatGPT, Perplexity)이 콘텐츠를 인식 못할 수 있음

### 비교: 한국어 랜딩(/ko)
whistle-landing.html은 React + Fade 컴포넌트를 사용하지만,
초기 렌더링 시 콘텐츠가 보임 (opacity 기본값 1).

## 수정 제안
```css
/* noscript/reduced-motion 대응 */
@media (prefers-reduced-motion: reduce) {
  .fade-up { opacity: 1 !important; transform: none !important; }
}
```

```html
<noscript>
<style>.fade-up { opacity: 1 !important; transform: none !important; }</style>
</noscript>
```

또는 Intersection Observer의 fallback:
```js
if (!('IntersectionObserver' in window)) {
  document.querySelectorAll('.fade-up').forEach(el => el.style.opacity = '1');
}
```
