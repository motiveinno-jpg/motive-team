# S8-bug-020: T() 함수 — 번역 키 노출 시 UX 영향 분석 (TASK-007 관련)

## 심각도: P0 (S2-bug-014, S3-bug-017과 동일 근본 원인)
## 페이지: whistle-app.html 전체 앱
## 유형: i18n + UX

## S8 관점 UX 영향 분석

### 근본 원인 (코드 확인됨)
`whistle-app.html:4786` — `T(key)` 함수의 최종 fallback이 `return key;`
→ 딕셔너리에 없는 키가 "export_intelligence", "no_analyses" 등으로 UI에 노출

### UX 영향 등급: **치명적**
1. **신뢰도 파괴**: 사용자가 코드 변수명을 보면 "이 서비스 완성 안 됨" 인식
2. **작업 불가**: 메뉴/버튼 텍스트가 코드명이면 기능을 찾을 수 없음
3. **글로벌 서비스 불가**: 13개 언어 지원 선언과 실제 불일치

### 현재 안전한 언어
- ko: ✅ `v[0]` 직접 반환
- en: ✅ `v[1]` 직접 반환
- 기타 11개 언어: ⚠️ _ML에 해당 키가 있어야만 번역됨

### 수정 제안 (S8 관점)
T() 함수 최종 fallback을 영어로 변경하면 즉시 해결:
```js
// 현재: return key;
// 수정: return v ? v[1] : (ml && ml['en']) || key;
```
이렇게 하면 번역이 없어도 최소한 영어가 표시되어 사용 가능.
