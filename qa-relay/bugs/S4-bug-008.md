# [S4-미국바이어] 사이드바 네비게이션이 제조사 앱으로 전환 | P1 | buyer-app.html

## 재현 경로
1. /app/buyer에서 바이어 계정으로 로그인
2. 대시보드 정상 표시
3. 사이드바에서 "Orders & Shipping" 또는 "Messages" 클릭
4. → /app (제조사 앱)으로 네비게이트되며, 다른 계정(Yamamoto Manufacturing Co., Ltd.)의 대시보드 표시

## 기대
- 바이어 앱 내에서 해당 섹션으로 라우팅

## 실제
- 사이드바 클릭 시 바이어 앱을 벗어나 제조사 앱으로 전환
- 다른 사용자(Yamamoto)의 세션으로 표시됨
- 언어도 일본어로 전환됨

## 근본 원인 추정
- 바이어 앱의 사이드바 onclick이 buyer-app 내부 라우팅 대신 제조사 앱 URL(/app)로 이동
- 또는 whistle-main.htm 라우터 프레임 내에서 바이어/제조사 앱이 공유 세션 사용
- Supabase auth 토큰이 교차 오염되어 다른 유저 세션 표시
