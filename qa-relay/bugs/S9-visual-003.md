## 신규 발견

### BUG-V021: 아랍어 RTL 레이아웃 미적용
- **심각도**: P2
- **뷰포트**: 데스크톱 + 모바일
- **페이지**: /app (아랍어 전환)
- **문제**: 아랍어로 전환하면 모든 텍스트는 아랍어로 정확히 번역되지만(تسجيل الدخول, البريد الإلكتروني, كلمة المرور 등), 레이아웃은 LTR(왼쪽→오른쪽) 그대로. 아랍어는 RTL(오른쪽→왼쪽)이 표준. 라벨이 왼쪽, 입력 필드 왼쪽 정렬 — 아랍 사용자에겐 "거울에 비친 것처럼" 느껴짐
- **스크린샷**: 3rd-arabic-login, 3rd-arabic-login-mobile
- **제안**: `dir="rtl"` 속성 + CSS `direction: rtl` 적용. 아랍어 선택 시 전체 폼 카드 반전

### BUG-V022: 영어 랜딩 모바일 — 바이어 매칭 카드 겹침
- **심각도**: P2
- **뷰포트**: 모바일 375px
- **페이지**: /en (바이어 매칭 섹션)
- **문제**: "Your Product" 카드와 바이어 매칭 결과 카드(GreenLife 94%, EuroTrade 87%, Sakura 82%)가 모바일에서 겹침. "Your Product" 텍스트가 "You Pro... Pre... Ski..." 형태로 잘림. 매칭 결과를 제대로 볼 수 없음
- **스크린샷**: 3rd-en-pricing-mobile (첫번째 모바일 캡처)
- **제안**: 모바일에서 "Your Product" → 매칭 결과를 세로 순서로 표시. 또는 매칭 카드만 보여주고 "Your Product"은 상단 요약으로

### BUG-V023: 영어 랜딩 — 히어로 수치 또 다른 불일치
- **심각도**: P3
- **뷰포트**: 데스크톱/모바일
- **페이지**: /en 히어로 vs 바이어 매칭 섹션
- **문제**:
  - 히어로: "200+ Countries Covered"
  - 바이어 매칭 섹션: "15+ Countries Covered"
  - 같은 페이지 안에서 국가 수가 200+ vs 15+로 13배 차이. 신뢰도 심각 훼손
- **스크린샷**: 02-landing-en-desktop vs 3rd-en-pricing-desktop
- **제안**: 통일. "200+"가 AI 분석 국가, "15+"가 바이어 실재 국가라면 문맥 표시 필요
