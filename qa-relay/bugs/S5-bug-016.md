# [S5-동남아바이어] P1: 로그인 후 세션 불안정 — 반복적 리다이렉트/로그아웃

- 심각도: P1 (사용 불가 수준)
- 뷰포트: 모바일 375x812 + 데스크톱 공통
- 페이지: /app/buyer 전체

## 문제
로그인 성공 후 페이지 이동 시 빈번하게 세션이 만료/소실됨.
- 바이어 앱에서 로그인 → 대시보드 정상 → 페이지 내 이동(Search→Deals 등) 시 갑자기 `/ko` 또는 `/buyer` 또는 `/admin`으로 리다이렉트
- `location.reload()` 후 세션 토큰이 사라지는 경우 빈번
- 5초 loadTimeout(buyer-app.html:2526) 안에 세션 확인 실패 시 로그아웃 처리

## 재현 빈도
- 10회 이상 로그인 시도 중 약 50% 실패율
- 특히 모바일 뷰포트 전환, viewport 리사이즈, 언어 변경 후 reload 시 높은 실패율

## 추정 원인
1. Supabase `getSession()` 호출이 5초 타임아웃 내에 완료 못 하는 경우
2. `sb.from('users').select()` 쿼리가 406 에러 반환 (RLS 정책 문제?)
3. loadUser() 라인 2556-2558의 role 확인 후 리다이렉트 로직
4. 브라우저 localStorage에 토큰 저장 후 Supabase 클라이언트가 인식 못 하는 경우

## 406 에러 로그 (콘솔)
로그인 후 매번 4~7건의 406 에러 발생:
```
Failed to load resource: the server responded with a status of 406 ()
```

## 제안
1. 5초 타임아웃 → 15초로 늘리거나 재시도 로직 추가
2. 406 에러 원인 조사 (Supabase RLS + Accept 헤더 불일치)
3. 세션 유효성 검증 실패 시 즉시 로그아웃 대신 재시도 1회
