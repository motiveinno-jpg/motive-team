# [S4-미국바이어] Sign Up 탭 클릭 시 제조사 가입 폼으로 전환 | P1 | buyer-app.html

## 재현 경로
1. https://whistle-ai.com/app/buyer 접속 (깨끗한 세션)
2. 영어 Sign In / Sign Up 탭 표시됨
3. Sign Up 클릭 (.auth-tab 클릭)
4. → 제조사 앱 (whistle-app.htm) 가입 폼이 로드됨
5. "輸出成功のスタート", "会社名", "無料で始める" 등 일본어 제조사 UI 표시
6. 바이어 가입 불가능

## 기대
- buyer-app.html 내부에서 바이어 가입 폼 (Company, Contact, Email, PW, Country, Interest, Create Account) 표시

## 실제
- .auth-tab 클릭이 buyer-app의 SPA 라우팅이 아닌 전체 페이지 네비게이션을 트리거
- /app/buyer → /app (whistle-app.htm)으로 이동
- 제조사 가입 폼이 일본어로 표시됨
- **바이어 신규 가입이 완전히 불가능**

## 근본 원인 추정
- buyer-app.html의 auth-tab onclick이 buyer-app의 B.set() 대신 whistle-app의 컨텍스트를 실행
- 또는 /app/buyer 경로에서 .auth-tab 클릭 시 /app/*의 wildcard 리다이렉트 규칙이 whistle-app.htm을 로드
- Cloudflare _redirects 규칙: /app/buyer → buyer-app.htm (200) vs /app/* → whistle-app.htm (200) — Sign Up 클릭이 hash/state 변경만 해야하는데 실제 네비게이션 발생

## 영향
- 바이어 신규 가입 100% 차단
- 기존 계정 로그인은 가능하나, 새 바이어 확보 불가
