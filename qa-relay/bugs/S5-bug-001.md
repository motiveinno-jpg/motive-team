# [S5-동남아바이어] P1: 언어 자동 감지 없음 — Accept-Language 무시

- 심각도: P1 (글로벌 서비스 첫인상 이탈)
- 뷰포트: 모바일 375x812
- 페이지: /app/buyer (전체)
- 페르소나: Nguyen Thi Mai (베트남 호치민, Accept-Language: vi-VN)

## 문제
Accept-Language: vi-VN,vi;q=0.9 헤더를 설정해도 베트남어 자동 전환 없음.
`buyer-app.html:431` — 코드가 `return 'en'` (무조건 영어 기본) 하드코딩.
`navigator.language`, `Accept-Language` 헤더 모두 무시.

## 현재 동작
1. 베트남 사용자 접속 → 영어(또는 한국어) 표시
2. 사용자가 직접 Profile > Language > Tiếng Việt 선택해야 함
3. 첫 접속 시 자동 감지 메커니즘 0%

## 코드 위치
```
buyer-app.html:425-432
var _userLang = (function() {
  var params = new URLSearchParams(location.search);
  var langParam = params.get('lang');
  if(langParam) { ... return langParam; }
  var saved = localStorage.getItem('whistle_lang');
  if(saved) return saved;
  return 'en'; // ← navigator.language 사용 안 함
})();
```

## 제안
```javascript
// navigator.language에서 언어 코드 추출
var browserLang = (navigator.language || '').substring(0,2).toLowerCase();
var supported = ['ko','en','ja','zh','vi','th','de','fr','es','pt','id','tr','ar'];
if(supported.indexOf(browserLang) >= 0) return browserLang;
return 'en';
```
