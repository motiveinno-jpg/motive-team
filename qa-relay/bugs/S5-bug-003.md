# [S5-동남아바이어] P1: 가입 시 429 Rate Limit — 에러 메시지 불친절

- 심각도: P1 (가입 차단)
- 뷰포트: 모바일 375x812
- 페이지: /app/buyer Sign Up

## 문제
1. 가입 시도 시 Supabase 이메일 rate limit(429) 발생
2. 에러 메시지: "email rate limit exceeded" — 영어 전문 용어, 초급자 이해 불가
3. 에러 발생 후 폼 데이터 전부 초기화됨 (다시 입력해야 함)

## 재현 순서
1. Sign Up 탭 클릭
2. 모든 필드 작성
3. Create Account 클릭
4. "email rate limit exceeded" 표시
5. 폼 데이터 전체 리셋

## 바이어 관점 (Nguyen Thi Mai)
- "email rate limit exceeded"가 뭔지 모름
- 내가 뭘 잘못했는지 모름
- 다시 모든 정보를 입력해야 해서 이탈

## 제안
1. 에러 메시지를 "Too many attempts. Please try again in a few minutes." 등 이해 가능한 문장으로
2. 다국어 에러 메시지 (vi: "Quá nhiều lần thử. Vui lòng thử lại sau vài phút.")
3. 에러 발생 시 폼 데이터 유지
4. Rate limit 대기 시간을 카운트다운으로 표시
