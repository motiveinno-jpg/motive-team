## P1 — 못 쓰겠다 (즉시 수정)

### BUG-V001: 바이어 랜딩 모바일 — 초기 로드 시 콘텐츠 겹침
- **심각도**: P1
- **뷰포트**: 모바일 375px
- **페이지**: /buyer (바이어 랜딩)
- **문제**: 페이지 첫 로드 시 검색 영역 위에 반투명 콘텐츠가 겹쳐 보임. 텍스트가 뒤엉켜 읽을 수 없고, 검색 버튼("제조사 찾기")도 다른 요소에 가려짐. 새로고침하면 해결되는 경우도 있어 로딩 타이밍/렌더링 순서 문제로 추정
- **스크린샷**: 03-buyer-landing-mobile
- **제안**: 검색 영역 위에 겹치는 요소의 z-index/position 확인. 로딩 순서 조정 필요

### BUG-V002: 한국어 랜딩 모바일 — CTA 버튼 텍스트 줄바꿈으로 읽기 어려움
- **심각도**: P1
- **뷰포트**: 모바일 375px
- **페이지**: / (한국어 랜딩 히어로)
- **문제**: "무료 수출 진단 받기" 버튼이 모바일에서 "무료 수출 진 / 단 받기"로 어색하게 줄바꿈됨. CTA의 의미가 한눈에 안 들어옴. "바우처로 무료 이용" 버튼도 일부 잘림
- **스크린샷**: 01-landing-ko-mobile
- **제안**: 모바일에서 CTA를 세로 스택으로 변경 (가로 나란히 → 세로 배치). 또는 텍스트를 "무료 분석 시작" 같이 짧게

### BUG-V003: 쿠키 배너가 모바일 CTA 영역 덮음
- **심각도**: P1
- **뷰포트**: 모바일 375px
- **페이지**: 모든 페이지
- **문제**: 하단 쿠키 동의 배너가 CTA 버튼, 요금제 카드 하단, 기타 중요 콘텐츠를 가림. 특히 랜딩 히어로에서 "무료 수출 진단 받기" 버튼과 겹침. 사용자가 먼저 쿠키를 처리하지 않으면 핵심 동작 불가
- **스크린샷**: 01-landing-ko-mobile, 03-buyer-mobile-top-recheck
- **제안**: 쿠키 배너를 상단 슬림 바(한 줄)로 변경하거나, 하단 고정 시 아래 콘텐츠에 padding-bottom 추가

## P2 — 불편하다 (빠른 시일 내 수정)

### BUG-V004: 한국어 랜딩 모바일 — 히어로 제목 줄바꿈 어색
- **심각도**: P2
- **뷰포트**: 모바일 375px
- **페이지**: / (한국어 랜딩)
- **문제**: "사진 한 장이면 수출이 시작됩니다"가 "수출이 시작됩 / 니다"로 줄바꿈. "됩니다"가 혼자 떨어져 시각적 무게감 깨짐
- **스크린샷**: 01-landing-ko-mobile
- **제안**: 모바일 font-size 축소 또는 "사진 한 장이면 / 수출이 시작됩니다"로 의도적 줄바꿈(`<br>`) 삽입

### BUG-V005: 서비스 미리보기 섹션 — 거대한 빈 공간
- **심각도**: P2
- **뷰포트**: 데스크톱 1280px
- **페이지**: /ko (서비스 미리보기 = "실제 서비스 미리보기" 아래)
- **문제**: "거래 완료!" 텍스트 아래에 약 400px 높이의 완전히 빈 흰 공간. 사용자가 "페이지가 깨졌나?" 또는 "아직 로딩 중인가?" 혼동. 스크롤을 멈추게 만드는 dead zone
- **스크린샷**: 01-landing-ko-desktop-pricing2
- **제안**: 빈 공간 제거하거나, 미리보기 콘텐츠를 채우거나, 섹션 높이를 컨텐츠에 맞게 축소

### BUG-V006: 요금제 카드 — 데스크톱에서 4개 카드 상단 잘림
- **심각도**: P2
- **뷰포트**: 데스크톱 1280px
- **페이지**: /ko (요금제 섹션)
- **문제**: 스크롤하면 요금제 카드 4개의 상단(플랜 이름, 가격)이 화면 위로 사라지고, 기능 목록만 보임. 어떤 플랜인지 모른 채 기능 비교를 하게 됨. sticky header가 없어서 플랜명을 기억해야 함
- **스크린샷**: 01-landing-ko-desktop-pricing4
- **제안**: 요금제 카드에 sticky 플랜명 헤더 추가, 또는 카드 높이를 줄여 한 화면에 들어오게

### BUG-V007: 영어 랜딩 모바일 — "Get Free Export Analysis" 버튼 텍스트 3줄 줄바꿈
- **심각도**: P2
- **뷰포트**: 모바일 375px
- **페이지**: /en (영어 랜딩)
- **문제**: CTA "Get Free / Export / Analysis"로 3줄 줄바꿈. 옆의 "Try Live / Demo"는 2줄. 버튼 높이가 들쭉날쭉. 깔끔하지 않음
- **스크린샷**: 02-landing-en-mobile
- **제안**: 모바일에서 "Free Analysis" + "Live Demo" 같이 짧게 줄이거나, 세로 스택으로 변경

### BUG-V008: 푸터 텍스트와 장식 라인 겹침
- **심각도**: P2
- **뷰포트**: 데스크톱 1280px
- **페이지**: /ko (푸터)
- **문제**: 푸터의 회사 정보 텍스트(사업자등록번호, 주소 등)와 위 섹션의 장식 라인이 겹침. 텍스트 가독성 떨어짐
- **스크린샷**: 01-landing-ko-footer-desktop
- **제안**: 장식 라인에 overflow: hidden 적용하거나, 푸터와의 간격 확보

### BUG-V009: 한국어 랜딩 — 4단계 프로세스 모바일 텍스트 빽빽함
- **심각도**: P2
- **뷰포트**: 모바일 375px
- **페이지**: /ko (4단계 프로세스: AI 분석→전략 수립→알리바바 입점→바이어 매칭)
- **문제**: 4개 단계가 한 줄에 모두 나오면서 텍스트가 극도로 밀집. "제품 사진/URL 업로드 → 13개 섹션 보고서" 같은 텍스트가 매우 작고, 줄바꿈이 단어 중간에서 발생. 숨 쉴 공간 없음
- **스크린샷**: 01-landing-ko-mobile-process
- **제안**: 모바일에서 4단계를 세로 리스트로 변환. 각 단계를 카드로 분리

## P3 — 어색하다 (여유 있을 때 개선)

### BUG-V010: 데스크톱 업로드 영역 — 점선 테두리가 산만
- **심각도**: P3
- **뷰포트**: 데스크톱 1280px
- **페이지**: /ko (제품 분석 입력 섹션)
- **문제**: 업로드 영역의 점선 테두리 + 격자 패턴이 시각적으로 산만. 깔끔한 다크 테마와 어울리지 않음. "뭔가 복잡해 보인다"는 첫인상
- **스크린샷**: 01-landing-ko-desktop-scroll1
- **제안**: 점선을 단색 실선(rgba)으로, 격자 패턴 제거

### BUG-V011: 한국어/영어 랜딩 — 수치 통계 불일치
- **심각도**: P3
- **뷰포트**: 데스크톱/모바일
- **페이지**: /ko vs /en
- **문제**: 한국어: "13개 분석 섹션, 2분 분석 완료". 영어: "13 Analysis Sections, 60s Analysis Time". 같은 서비스인데 2분 vs 60초 — 수치가 다름. 신뢰 깨짐
- **스크린샷**: 01-landing-ko-desktop vs 02-landing-en-desktop
- **제안**: 통일. 실제 측정 기준으로 하나의 수치만 사용

### BUG-V012: 바이어 랜딩 — "아시아+" 통계가 의미 불분명
- **심각도**: P3
- **뷰포트**: 데스크톱/모바일
- **페이지**: /buyer
- **문제**: "아시아+ / 글로벌 확장 중"이라는 통계가 숫자도 아니고 구체적이지 않음. 다른 통계(AI, Stripe, 무료)는 명확한데 이것만 모호. "뭘 의미하는지 모르겠다"
- **스크린샷**: 03-buyer-landing-desktop
- **제안**: "31개국" 또는 "200+ 국가 분석" 같이 구체적 숫자로

### BUG-V013: 로그인 페이지 — 14개 언어 버튼이 압도적
- **심각도**: P3
- **뷰포트**: 데스크톱/모바일
- **페이지**: /app (로그인)
- **문제**: 로그인 폼 아래에 14개 언어 버튼이 3줄로 배치. 로그인이 목적인 페이지에서 시선 분산. 특히 모바일에서 언어 버튼이 폼보다 큰 공간 차지
- **스크린샷**: 04-login-desktop, 04-login-mobile
- **제안**: 언어 선택을 드롭다운/셀렉트로 변경. 또는 상단에 작은 아이콘으로

### BUG-V014: 영어 랜딩 — 국가 카드(India, Vietnam, UAE) 세로 쌓임
- **심각도**: P3
- **뷰포트**: 모바일 375px
- **페이지**: /en (국가별 인증 섹션)
- **문제**: 데스크톱에서 5개 나란히 보이는 국가 카드가 모바일에서 세로로 쌓이면서 스크롤이 매우 길어짐. 5개 카드를 지나치려면 약 5~6번 스와이프
- **스크린샷**: 02-landing-en-mobile-scroll1
- **제안**: 모바일에서 가로 스크롤 카루셀로 변경

### BUG-V015: 영어 랜딩 — "Voucher" 메뉴 한국어에만 해당
- **심각도**: P3
- **뷰포트**: 데스크톱 1280px
- **페이지**: /ko 네비게이션
- **문제**: 한국어 네비에 "바우처" 메뉴가 있는데 영어 네비에는 없음. 일관성은 괜찮지만, 한국어 "바우처" 옆의 빨간 dot(알림 표시)이 무엇을 의미하는지 설명 없음
- **스크린샷**: 01-landing-ko-desktop
- **제안**: dot에 마우스오버 시 툴팁 추가 ("바우처 신청 가능")

## 체크리스트 현황

### 데스크톱 (1280px+)
- [x] 텍스트 겹침/잘림 → P2 푸터
- [x] 버튼 정렬 → 양호
- [x] 색상/대비 → 양호 (다크 테마 일관)
- [x] 폰트 크기 일관성 → 양호
- [x] 여백/간격 들쭉날쭉 → P2 빈 공간
- [x] 아이콘/텍스트 정렬 → 양호
- [x] 빈 공간 어색 → P2 (서비스 미리보기)
- [x] 레이아웃 점프 → 미확인 (정적 스크린샷 한계)
- [x] 헤더 겹침 → 양호 (sticky nav 정상)
- [x] 가로 스크롤바 → 없음 (양호)
- [x] 이미지 찌그러짐 → 없음
- [x] 테이블/카드 화면 밖 → P2 요금제 잘림

- [x] "뭘 해야 하는지 모르겠다" → 양호 (CTA 명확)

### 모바일 (375px) — 더 중요
- [x] 터치 타겟 너무 작음 → 양호 (버튼 충분히 큼)
- [x] 텍스트 너무 작음 → P2 (4단계 설명)
- [x] 모달 화면 넘어감 → 미확인
- [x] 키보드로 가려짐 → 미확인 (스크린샷 한계)
- [x] 좌우 스크롤 → 없음 (양호)
- [x] 메뉴 한 손 탐색 → 양호 (햄버거 메뉴)
- [x] 스크롤 지옥 → P3 (국가 카드 세로 쌓임)
- [x] 이미지 원본 크기 → 없음
- [x] 하단 내비게이션 없음 → 랜딩이라 해당 없음
- [x] 가로 모드 붕괴 → 미확인
