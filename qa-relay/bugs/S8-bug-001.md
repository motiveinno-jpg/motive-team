# S8-bug-001: 색상 대비 접근성 위반 (WCAG 2.2 AA) — 한국어 랜딩

## 심각도: P1
## 페이지: whistle-ai.com/ko (whistle-landing.html)
## 유형: 접근성

## 설명
색상 대비 검사에서 50건 이상의 WCAG AA 위반 발견. 대부분 배지(badge)/태그 요소에서 텍스트 색상과 배경색이 동일하게 감지됨.

## 주요 위반 케이스

### 1. 네비게이션 버튼 — 대비 부족
- "서비스", "요금제", "바우처", "제휴문의", "로그인" 버튼: `rgba(255,255,255,0.6)` on dark bg
- 대비율: ~6.8:1 (이건 AA 통과하지만, 0.6 opacity라 실제 렌더링에서 더 낮을 수 있음)

### 2. 배지/태그 텍스트 — 대비 1:1 (심각)
- "🤖 AI 수출 분석 엔진" 배지: cyan(#00D4FF) on cyan(#00D4FF) — ratio 1:1
- "🏷️ 수출바우처" 배지: gold(#FFB800) on gold(#FFB800) — ratio 1:1
- "무료 체험" 배지: 동일 문제
- **원인**: 배지들이 border + transparent bg 스타일인데 대비 검사 알고리즘이 border 색을 bg로 감지. 실제 시각적으로는 읽히지만, 스크린리더+저시력자에게 문제 가능

### 3. 딜 타임라인 버튼 (2~10단계) — 비활성 상태 대비 부족
- "견적서 발송", "샘플 발송", "협상 & 계약" 등: `rgb(0,0,0)` on `rgb(13,20,36)` → 대비 1.14:1
- **실제 문제**: 비활성 탭의 텍스트가 거의 보이지 않음

### 4. "협력사" 텍스트
- `rgb(168,85,247)` (보라) on white 계열 → 대비 3.96:1 (AA 기준 4.5:1 미달)

## 재현 방법
1. whistle-ai.com/ko 접속
2. 서비스 프리뷰 섹션의 딜 타임라인 확인 (비활성 단계)
3. 국가별 규정 섹션의 배지 텍스트 확인

## 수정 제안
1. 비활성 탭 텍스트를 최소 `rgba(255,255,255,0.45)` 이상으로 변경
2. 배지 요소에 명시적 배경색 추가 (반투명 배경이라면 충분한 대비 확보)
3. 보라색 "협력사" → 더 밝은 보라 또는 명도 조정

## DEFERRED (2026-03-30)
- 비활성 탭/배지 대비 이슈는 whistle-landing.html(React) CSS 디자인 변경 필요
- 배지 1:1 대비는 테스트 오탐 가능성 (border color ≠ background color)
- 비활성 탭 텍스트 대비는 실제 이슈 → 디자인 리뷰 후 수정 예정
