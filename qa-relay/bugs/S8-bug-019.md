# S8-bug-019: 아랍어 — RTL 레이아웃 미적용 + htmlLang 불일치

## 심각도: P1
## 페이지: whistle-ai.com/app?lang=ar (제조사앱)
## 유형: 접근성 + i18n + UX

## 설명
### 1. RTL(Right-to-Left) 레이아웃 미적용
아랍어 선택 시 텍스트는 아랍어로 번역되지만 페이지 방향이 LTR 그대로:
- `<html dir="ltr">` — RTL로 변경 필요
- `direction: ltr` (CSS) — `rtl` 필요
- 라벨, 버튼, 입력 필드 모두 왼쪽 정렬 — 아랍어 사용자에겐 부자연스러움

### 2. htmlLang 불일치
- 아랍어 선택했는데 `<html lang="en">` — `lang="ar"` 이어야 함
- 스크린리더가 영어 발음으로 아랍어 텍스트를 읽으려 시도

### 3. 번역 품질은 우수
- "تسجيل الدخول" (로그인), "إنشاء حساب" (회원가입)
- "البريد الإلكتروني" (이메일), "كلمة المرور" (비밀번호)
- "بداية نجاح التصدير" (수출 성공의 시작)
- 13개 언어 선택기에 العربية 🇸🇦 올바르게 표시

## 영향
- 중동 사용자(UAE, 사우디, 쿠웨이트 등) 전체
- 타겟 국가 31개 중 UAE, 사우디 포함

## 수정 제안
```js
// 언어 설정 시 RTL 감지 추가
var RTL_LANGS = ['ar', 'he', 'fa', 'ur'];
if (RTL_LANGS.indexOf(_userLang) >= 0) {
  document.documentElement.setAttribute('dir', 'rtl');
} else {
  document.documentElement.setAttribute('dir', 'ltr');
}
document.documentElement.setAttribute('lang', _userLang);
```

CSS에 RTL 기본 지원 추가:
```css
[dir="rtl"] .sidebar { right: 0; left: auto; }
[dir="rtl"] .topbar { flex-direction: row-reverse; }
[dir="rtl"] input, [dir="rtl"] select { text-align: right; }
```
