- 심각도: P2(불편)
- 발견 시각: 2026-03-29 23:50
- 페이지: /app (전체 — 쿠키 배너 공통)
- 관련 버그: S2-bug-005-reopen
- 재현 경로:
  1. localStorage에 whistle_lang='ja' 설정
  2. /app 접속
  3. 쿠키 배너 확인 → 영어로 표시
- 근본 원인:
  - whistle-app.html line 23963: `var _lang=typeof _userLang!=='undefined'?_userLang:'en';`
  - _userLang은 메인 앱 코드(line ~150)에서 정의됨
  - 그러나 쿠키 배너 IIFE(line 23961)는 페이지 하단에 있어 DOM 순서상 나중에 실행
  - 문제: _userLang 정의가 메인 앱 `<style>` 블록 다음이 아니라 `<script>` 내부에서 이루어짐
  - 쿠키 배너 시점에 `typeof _userLang === 'undefined'` → 'en' fallback
- 검증 데이터:
  - `_userLang`: "undefined" (IIFE 실행 시)
  - `document.documentElement.lang`: "en" (쿠키 배너 시점)
  - 하지만 앱 렌더링 후에는 `_userLang = 'ja'`로 설정됨
- 해결 방안:
  1. 쿠키 배너 IIFE에서 `_userLang` 대신 `localStorage.getItem('whistle_lang')` 직접 읽기
  2. 또는 쿠키 배너를 DOMContentLoaded 후 실행
- 스크린샷: 확인됨 (쿠키 배너 영어, 나머지 UI 일본어)
- 상태: ✅ ALREADY FIXED — 쿠키 배너 IIFE에 robust fallback chain 적용됨: _userLang → URL param → localStorage → navigator.language. _userLang은 파일 상단에서 전역 정의되어 IIFE 실행 시점에 이미 사용 가능
