# S8-bug-011: 바이어 랜딩 — 랜드마크/ARIA 전무 + 언어 혼용

## 심각도: P1
## 페이지: whistle-ai.com/buyer (buyer.html 추정)
## 유형: 접근성 + UX

## 설명
### 접근성
- nav: 0, main: 0, header: 0, footer: 0
- ARIA 속성: 0개
- 한국어 랜딩(19개), 영어 랜딩(3개)과 비교해 완전 누락

### 언어 혼용 문제
- `lang="ko"` 설정인데 H2에 영어 "See It In Action" 혼용
- 한/영 혼용 헤딩은 스크린리더가 잘못된 발음으로 읽음
- 해결: `<h2 lang="en">See It In Action</h2>` 또는 한국어로 번역

### 성능: 양호
- TTFB 363ms, 로드 620ms — 매우 빠름

## 수정 제안
1. 시맨틱 랜드마크 추가 (nav, main, footer)
2. "See It In Action" → 한국어로 번역 또는 lang="en" 속성 추가
3. 검색 입력 필드에 aria-label 추가
