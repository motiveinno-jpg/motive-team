# QA Relay — 세션 간 공유 디렉토리

## 구조
- `bugs/` — 각 세션이 발견한 버그 (S1-bug-001.md, S2-bug-001.md ...)
- `fixes/` — S4(개발)가 수정한 내역 (S4-fix-001.md ...)
- `intel/` — S6(인텔리전스)가 조사한 트렌드/인사이트
- `reports/` — 각 세션 진행 상황 보고

## 파일 네이밍 규칙
- `S{번호}-{타입}-{순번}.md` (예: S1-bug-001.md, S6-intel-001.md)
- 각 파일 첫 줄: `# [S{번호}-{역할명}] {제목}`
