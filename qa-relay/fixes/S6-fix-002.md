# S6-fix-002: S1-bug-001 수정 — 회원가입 에러 메시지 미표시

## 버그ID
- S1-bug-001: 회원가입 실패 시 에러 메시지 미표시

## 원인
회원가입 에러 처리에 `alert()` 사용. 브라우저에 따라 alert 차단 가능,
특히 headless 브라우저/자동화 환경에서 피드백 없음.
또한 error.message가 undefined일 때 `error.message.includes()` TypeError 가능.

## 수정파일
- whistle-landing.html

## 수정내용
1. `signupError` 상태 변수 추가
2. `handleSignup()` 내 모든 `alert()` → `setSignupError()` 교체
3. 에러 메시지 한/영 이중언어 대응 (`L()` 함수 사용)
4. `error.message && error.message.includes()` 패턴으로 TypeError 방지
5. "email" 관련 Supabase 에러에 대한 추가 매칭 패턴
6. 모달 UI에 `role="alert"` 인라인 에러 표시 영역 추가
7. `openSignup()` 시 에러 초기화

## 영향범위
- /ko 랜딩 회원가입 모달 전체
- 기존 기능에 영향 없음 (alert → inline 에러로 UX 개선)
