# S6-fix-007: buyer-app.html 언어 자동 감지 추가 + TASK-001 분석

## 버그ID
- S2-bug-008 (부분): 언어 자동 감지 실패
- S3-bug-008 (부분): Accept-Language 미사용

## 수정파일
- buyer-app.html

## 수정내용
buyer-app.html의 `_userLang` 초기화에 navigator.language 기반 자동 감지 추가.
기존: `return 'en'` (항상 영어 기본값)
변경: supported 언어 리스트 확인 → navigator.language 매칭

## TASK-001 언어 자동 감지 현황 분석

| 페이지 | 경로 | 자동감지 | 방법 | 지원 언어 |
|--------|------|----------|------|-----------|
| 메인 랜딩 | / | O | i18n-detect.js (navigator.language + CF country) | 13개 |
| 한국어 랜딩 | /ko | O (ko만) | inline script | ko, en |
| 영어 랜딩 | /en | X (en 고정) | 정적 | en |
| 제조사 앱 | /app | O | inline (navigator.language) | 13개 |
| 바이어 앱 | /app/buyer | O **(수정됨)** | inline (navigator.language) | 13개 |
| 관리자 | /admin | O | whistle-app.html과 동일 | 13개 |

### Accept-Language 헤더 관련
- Accept-Language는 서버사이드(HTTP 헤더)이므로 클라이언트 JS에서 직접 읽을 수 없음
- Cloudflare Workers에서 `CF-IPCountry` 헤더를 meta 태그로 주입하면 사용 가능
- 현재: i18n-detect.js가 `meta[name="cf-ipcountry"]` 체크하지만 Workers 미설정
- navigator.language가 유일한 실질적 감지 수단 → 정상 동작 확인

### 결론
navigator.language 기반 감지는 모든 주요 페이지에서 동작.
Accept-Language 헤더 기반 감지는 Cloudflare Workers 설정이 필요하며 별도 작업.
