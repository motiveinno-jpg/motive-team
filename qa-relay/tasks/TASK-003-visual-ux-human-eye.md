# [S0→S1,S2,S3,S4,S5,S8] 시각적 UX — 사람 눈으로 봤을 때 불편한 것 전부 찾아라

## 우선순위: P1

## 지시
코드/접근성 감사가 아니라, **실제 사용자가 화면을 봤을 때 느끼는 불편함**을 찾아라.
Puppeteer 스크린샷을 찍고, 그 스크린샷을 **사람의 눈으로 보듯이** 판단해라.

## 데스크톱 (1280px+) 체크리스트
- [ ] 텍스트가 겹치거나 잘리는 곳
- [ ] 버튼이 너무 작거나, 너무 크거나, 정렬 안 맞는 곳
- [ ] 색상이 눈에 피로한 곳 (너무 밝은 배경에 밝은 텍스트, 대비 부족)
- [ ] 폰트 크기가 갑자기 달라지는 곳 (일관성 깨짐)
- [ ] 여백/간격이 들쭉날쭉한 곳
- [ ] 아이콘과 텍스트 수직 정렬 안 맞는 곳
- [ ] 빈 공간이 어색하게 넓은 곳
- [ ] 로딩 중 화면 깜빡임이나 레이아웃 점프
- [ ] 스크롤 시 헤더/사이드바 겹침
- [ ] 가로 스크롤바가 나타나는 곳
- [ ] 이미지가 찌그러지거나 흐린 곳
- [ ] 테이블/카드가 화면 밖으로 나가는 곳
- [ ] 그래프/차트 라벨 겹침
- [ ] 한국어/영어/일본어 전환 시 레이아웃 밀리는 곳
- [ ] "뭘 해야 하는지 모르겠다"는 느낌이 드는 페이지

## 모바일 (375px) 체크리스트 — 더 중요
- [ ] 터치 타겟이 너무 작아서 손가락으로 누르기 어려운 버튼
- [ ] 텍스트가 너무 작아서 읽기 힘든 곳 (14px 미만)
- [ ] 모달/팝업이 화면을 넘어가는 곳
- [ ] 입력 폼에 키보드 올라오면 버튼/콘텐츠가 가려지는 곳
- [ ] 좌우 스크롤이 생기는 곳
- [ ] 메뉴가 한 손으로 탐색 불가능한 곳
- [ ] 카드/목록이 세로로 너무 길어져서 스크롤 지옥
- [ ] 이미지가 모바일에서 원본 크기로 표시 (깨짐/느림)
- [ ] 하단 내비게이션 없음 → 매번 위로 올라가야 함
- [ ] 가로 모드(landscape) 전환 시 레이아웃 붕괴

## 테스트 방법
1. 각 페이지에서 데스크톱(1280px) + 모바일(375x812) 스크린샷 둘 다 촬영
2. 스크린샷을 보고 "이 화면을 처음 보는 사람이 불편하겠다"는 것을 기록
3. 기술적 이유 말고, **사람의 느낌**으로 판단:
   - "여기 뭔가 어색하다"
   - "이건 못 누르겠다"
   - "이게 뭔지 모르겠다"
   - "너무 빽빽하다"
   - "너무 허전하다"
   - "글씨가 안 보인다"

## 테스트 대상 페이지 (순서대로)
1. 한국어 랜딩 (/) — 데스크톱 + 모바일
2. 영어 랜딩 — 데스크톱 + 모바일
3. 바이어 랜딩 (/buyer) — 데스크톱 + 모바일
4. 로그인/가입 화면 — 데스크톱 + 모바일
5. 제조사 대시보드 — 데스크톱 + 모바일
6. 수출 분석 결과 — 데스크톱 + 모바일
7. 서류 생성/프리뷰 — 데스크톱 + 모바일
8. 채팅 — 데스크톱 + 모바일
9. 바이어 대시보드 — 데스크톱 + 모바일
10. 설정/프로필 — 데스크톱 + 모바일
