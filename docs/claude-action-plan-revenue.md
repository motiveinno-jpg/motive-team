### PHASE 1: 수익 개통 (이번주)

#### 1-1. 서류 과금 시스템 구현
```javascript
// 서류 생성 시 billing_event 자동 생성
function trackDocBilling(docType, dealId) {
  const prices = { pi:5000, ci:5000, pl:3000, co:10000, sc:15000, bl:5000 };
  sb.from('billing_events').insert({
    user_id: S.user.id,
    event_type: 'document_generation',
    feature: 'document_' + docType,
    amount: prices[docType] || 5000,
    currency: 'KRW',
    deal_id: dealId,
    status: 'pending'
  });
}
```
- 무료 플랜: 월 1건 무료, 이후 건당 과금
- Pro 플랜: 무제한 포함
- 관리자 대시보드에서 미수금 조회 가능

#### 1-2. 분석 리포트 과금 강화
```
현재: 플랜별 월간 제한 (free:3, starter:10, pro:무제한)
추가:
- 프리미엄 분석 (3인 전문가 패널) → 건당 300,000원
- 기본 분석 → 건당 50,000원 (무료 초과분)
- 수출바우처로 결제 가능 안내
```

#### 1-3. 에스크로 결제 UI 구현
- 바이어 → 에스크로 입금 (Stripe Connect)
- 마일스톤별 자동 릴리즈 (생산완료/선적/도착)
- 플랫폼 수수료 자동 차감 (1.5~5%)
- **이미 스키마 완비** → UI + Edge Function만 추가

### PHASE 2: 바우처 수익 자동화 (이번달)

#### 2-1. 수출바우처 결과보고 자동화
```
서비스 수행 완료 감지 (deal stage = completed)
  → 사업결과보고서 자동 생성
     - 서비스 수행 내역 (analyses + documents 이력)
     - 수출 성과 지표 (바이어 접촉수, 견적수, 계약수)
     - 전자세금계산서 연동 (Toss 발행 내역)
     - 결과물 증빙 자동 첨부 (PDF 리포트 + 서류)
  → PDF 다운로드 → 수행기관 정산 요청
```

#### 2-2. 바우처 자격 진단 + 수혜기업 만들기
```
입력: 사업자번호, 매출액, 수출실적
→ AI 자동 진단:
  - 수출바우처 자격 (Y/N + 부족 항목)
  - 혁신바우처 자격 (Y/N)
  - AI바우처 자격 (Y/N)
→ 미달 시 보완 계획 제시 (모든 보완 비용 = 휘슬 서비스 사용)
```

### PHASE 3: 글로벌 확장 준비 (Q2)

#### 3-1. i18n 프레임워크 도입
```javascript
// 단일 HTML SPA에서 i18n 구현 (라이브러리 없이 경량)
const T = {
  ko: { 'nav.dashboard': '대시보드', 'nav.analysis': 'AI 수출분석', ... },
  en: { 'nav.dashboard': 'Dashboard', 'nav.analysis': 'AI Export Analysis', ... },
  ja: { 'nav.dashboard': 'ダッシュボード', ... },
  vi: { 'nav.dashboard': 'Bảng điều khiển', ... }
};
function t(key) { return T[S.lang]?.[key] || T['ko'][key] || key; }
```
- 모든 UI 텍스트를 t() 함수로 감싸기
- 사이드바에 언어 전환기 추가
- 초기 지원: ko, en, ja, vi (한국, 영어, 일본, 베트남)

#### 3-2. 다통화 지원
```
- 기본 통화: KRW
- 지원 통화: USD, JPY, EUR, VND, THB
- 환율 API 연동 (실시간)
- 서류/분석/에스크로 모두 선택 통화로 표시
```

#### 3-3. 다법률/인증 대응 구조
```
국가별 필요 인증 DB:
- 한국 → 미국: FDA, FCC, UL
- 한국 → EU: CE, RoHS, REACH
- 한국 → 일본: PSE, JFRL, JIS
- 한국 → 동남아: SIRIM, TIS, SNI
→ AI 분석 시 타겟 국가별 인증 목록 자동 표시
```
