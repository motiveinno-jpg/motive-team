HIGH  H1 — CSP 'unsafe-inline'/'unsafe-eval' 제거

Content Security Policy에서 nonce 방식으로 전환. 인라인 스크립트 모두 제거 필요.

⚠️ 사유: Cloudflare Worker로 nonce 주입 필요 — 전체 인프라 아키텍처 변경. Vanilla JS SPA 구조상 단기 해결 불가.

HIGH  H2 — CSRF 토큰

partner-apply, 랜딩 폼 전송에 CSRF 보호 추가.

⚠️ 사유: Supabase 세션 + SameSite=Strict 쿠키로 실질적 보호됨. 추가 구현은 서버사이드 상태 관리 필요.

HIGH  H6 — 파트너 신청 민감 데이터 암호화

사업자번호, 연락처 등 민감 데이터 서버사이드 처리.

⚠️ 사유: 현재 RLS 적용된 Supabase 테이블에 저장. 필드 레벨 암호화는 EF 리팩터링 필요.

HIGH  H11 — 바이어 매칭 시드 데이터

주요 HS코드별 수입업체 데이터 투입. 현재 ai_generated 소스로 20개 시드 삽입.

⚠️ 사유: Apollo.io API + bl-match-buyers EF 정상 작동. 실제 바이어 데이터는 운영 과정에서 자연 증가 예정.

HIGH  H20 — 13개 언어 전체 번역

현재 한/영 완성. 중국어/일본어 등 13개 언어 실제 콘텐츠 번역 + RTL CSS.

⚠️ 사유: 대규모 번역 작업 (~수천 문자열). 글로벌 런칭 후 수요 높은 언어부터 순차 추가 권장.

MEDIUM  M12 — 알리바바 페이지 다국어

알리바바 연동 페이지 한국어 하드코딩 → _ML 함수 전환.

⚠️ 사유: 알리바바 콘텐츠는 ko_made에서만 사용 (한국 제조사 전용). 우선순위 낮음.

MEDIUM  M15 — 온보딩 실제 앱 스크린샷

onboarding.html 3단계 카드에 실제 앱 UI 스크린샷 삽입.

⚠️ 사유: 스크린샷 자산 필요. Puppeteer 자동 캡처 또는 수동 제작 후 추가 예정.

MEDIUM  M17 — HSTS Preload 등록

hstspreload.org에 whistle-ai.com 등록. 브라우저 레벨 HTTPS 강제.

⚠️ 사유: 등록 후 삭제 어려움 (6~12개월 소요). CEO 최종 승인 후 진행.
