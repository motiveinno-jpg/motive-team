## 1. 정산 & 환불 자동화

### 1.1 현재 상태

| 기능 | 구현 | 자동화 |
|------|------|--------|
| 에스크로 결제 (Stripe manual capture) | ✅ | 수동 |
| 7일 자동 정산 (auto-settle EF) | ✅ | ⚠️ 크론 미연결 |
| 구독 환불 (7일 쿨링) | ✅ | 수동 요청 |
| 에스크로 분쟁 환불 | ✅ | 수동 (관리자) |
| 정산 알림 (셀러) | ✅ | 자동 |
| 미수락 리마인더 | ❌ | 미구현 |
| 출하 지연 알림 | ❌ | 미구현 |
| 14일 자동 수취 확인 | ❌ | 미구현 |

### 1.2 자동화 구현 계획

#### A. 크론 기반 자동 정산 (n8n)

```
워크플로우: auto-settle-cron
트리거: 매일 06:00 KST
플로우:
  [크론] → [HTTP POST: auto-settle EF (x-cron-secret 헤더)]
         → [IF: settled > 0]
           → [텔레그램: "N건 자동 정산 완료, 총 $X"]
         → [IF: failed > 0]
           → [텔레그램: "⚠️ N건 정산 실패 — 수동 확인 필요"]
```

#### B. 주문 미수락 자동 취소 + 리마인더

```
워크플로우: order-acceptance-monitor
트리거: 매 3시간
플로우:
  [크론] → [Supabase: payments WHERE status='pending' AND type='escrow']
         → [반복: 각 건]
           → [계산: 영업일 경과]
           → [IF: 1~2일 경과]
             → [알림: 셀러에게 "주문 수락 리마인더"]
           → [IF: 3일 초과]
             → [Stripe: refund 전액]
             → [Supabase: status='auto_canceled']
             → [알림: 바이어 "자동 취소 + 환불", 셀러 "미수락 자동 취소"]
             → [텔레그램: CEO 알림]
```

#### C. 출하 지연 모니터링

```
워크플로우: shipment-delay-monitor
트리거: 매일 09:00 KST
플로우:
  [크론] → [Supabase: matchings WHERE status='accepted' AND shipped_at IS NULL]
         → [반복: 각 건]
           → [계산: 수락 후 경과일]
           → [IF: 3일 경과]
             → [알림: 셀러 "출하 리마인더"]
           → [IF: 5일 초과]
             → [알림: 바이어 "출하 지연 — 연장/취소 선택"]
             → [텔레그램: CEO 알림]
```

#### D. 14일 자동 수취 확인

```
워크플로우: auto-delivery-confirm
트리거: 매일 06:00 KST
플로우:
  [크론] → [Supabase: matchings WHERE status='shipped'
           AND delivered_at IS NULL
           AND shipped_at < NOW() - 14 days]
         → [반복: 각 건]
           → [Supabase: UPDATE delivered_at=NOW(), buyer_confirmed_at=NOW()]
           → [알림: 바이어 "14일 경과 자동 수취 확인"]
           → [알림: 셀러 "자동 수취 확인 — 7일 후 정산"]
         → [텔레그램: "N건 자동 수취 확인"]
```

#### E. 정산 리포트 (CEO 대시보드)

```
워크플로우: settlement-daily-report
트리거: 매일 22:00 KST
플로우:
  [크론] → [Supabase: 오늘 정산/환불 집계]
         → [Stripe: Balance + Pending 조회]
         → [텔레그램 리포트:]
           💰 정산 리포트 (2026-03-22)

           오늘 정산: 2건 / $1,240
           대기 중: 5건 / $3,200
           환불 처리: 0건
           미수락 알림: 1건
           출하 지연: 0건

           Stripe 잔액: $4,500
           이번 달 총 정산: $12,300
           이번 달 수수료 수입: $307.50
```

## 2. CS 자동화

### 2.1 티어별 처리

```
Tier 1 (즉시 자동 응답) — 80% 커버:
├── FAQ 봇: 가격, 기능, 사용법, 결제 방법
├── 상태 조회: "내 주문 상태", "정산 언제?"
├── 가이드 링크: 수출 서류, HS코드, FTA
└── 언어 감지: 한/영/중/일 자동 전환

Tier 2 (AI 분석 후 응답) — 15% 커버:
├── 에스크로 분쟁 초기 접수
├── 기술 문의 (분석 결과 해석)
├── 계정 문제 (비밀번호, 플랜)
└── 버그 리포트 접수 + 자동 이슈 생성

Tier 3 (CEO 에스컬레이션) — 5%:
├── 환불 분쟁 (금액 큰 건)
├── 파트너십 문의
├── 법적 문의
└── VIP 고객 불만
```

### 2.2 구현 방법

```
워크플로우: cs-auto-responder
트리거: Supabase Realtime (messages 테이블 INSERT)
플로우:
  [새 메시지] → [분류: FAQ/상태조회/기술/분쟁/기타]
              → [Switch]
                ├→ [FAQ] → [Claude: FAQ DB에서 답변 생성] → [자동 발송]
                ├→ [상태조회] → [Supabase: 주문/정산 조회] → [답변 생성] → [발송]
                ├→ [기술] → [Claude: 상세 분석] → [초안 생성] → [검토 큐]
                ├→ [분쟁] → [증거 수집 안내] → [관리자 큐]
                └→ [기타] → [텔레그램: CEO 알림]
              → [응답 시간 로그]
              → [만족도 조사 (24시간 후)]
```

## 4. 개발 & 버그 자동화

### 4.1 에러 감지 → 자동 수정 파이프라인

```
워크플로우: error-auto-fix
트리거: 에러 로그 감지 (Supabase Edge Function logs)

[에러 감지] → [분류: 심각도 P1/P2/P3]
            → [P1 (서비스 다운)]
              → [텔레그램: 즉시 CEO 알림]
              → [PC1: Claude Code가 자동 진단]
              → [IF: 자동 수정 가능]
                → [수정 PR 생성] → [테스트] → [대표님 승인 대기]
              → [IF: 수동 필요]
                → [진단 결과 + 제안 텔레그램 발송]
            → [P2 (기능 오류)]
              → [GitHub Issue 자동 생성]
              → [PC1: 분석 + 수정 시도]
              → [일일 리포트에 포함]
            → [P3 (경미)]
              → [로그 기록만]
              → [주간 리포트에 포함]
```

### 4.2 자동 QA 파이프라인

```
워크플로우: daily-qa
트리거: 매일 03:00 KST (트래픽 최저)

[크론] → [PC2: browse로 주요 경로 E2E 테스트]
        ├→ 로그인 → 대시보드 로드
        ├→ AI 분석 실행
        ├→ 프로젝트 생성 → 스테이지 전환
        ├→ 서류 생성
        ├→ 바이어 앱 로그인 → 검색 → 문의
        └→ 관리자 포털 접근
       → [결과 수집]
       → [IF: 실패 있음]
         → [스크린샷 + 에러 로그]
         → [텔레그램: CEO 알림]
         → [GitHub Issue 자동 생성]
       → [일일 QA 리포트 저장]
```
