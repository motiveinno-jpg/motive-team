2. 오류별 상세 대응

오류 유형 1

Stripe API 오류

원인: Stripe 서버 장애, 네트워크 타임아웃, API 키 만료, 요청 한도 초과 (429)

감지 방법:

  • Stripe 웹훅 응답 코드 모니터링 (4xx, 5xx)
  • 정산 큐에서 failed 상태 건수 알림
  • Supabase 로그에서 Edge Function 오류 추적

대응 절차:

  1. Stripe Status Page (status.stripe.com) 확인
  2. API 키 유효성 확인 (Stripe Dashboard > Developers)
  3. 자동 재시도 로직 작동 확인 (최대 3회, 지수 백오프)
  4. 3회 실패 시 정산 큐에서 수동 재실행

에스컬레이션: 1시간 이상 지속 시 Stripe Support 연락

오류 유형 2

캡처 실패

원인: Authorization Hold 만료 (7일 초과), 카드 잔액 부족, 카드 해지/분실, 통화 불일치

감지 방법:

  • 웹훅 이벤트 payment_intent.capture_failed 수신
  • 정산 스케줄러에서 캡처 실패 건 자동 플래그
  • 어드민 대시보드 "캡처 실패" 알림

대응 절차:

  1. 실패 원인 확인 (Stripe Dashboard > Payments > 해당 건)
  2. Authorization Hold 만료인 경우: 바이어에게 재결제 요청 알림 발송
  3. 카드 문제인 경우: 바이어에게 결제 수단 변경 요청
  4. 72시간 내 미해결 시 거래 보류 및 양측 통보

에스컬레이션: 72시간 미해결 시 CEO 보고 + 수동 처리

오류 유형 3

이중 정산

원인: 웹훅 중복 수신, 재시도 로직 버그, 동시성 문제 (Race Condition)

감지 방법:

  • 정산 테이블에서 동일 payment_intent_id에 대한 중복 레코드 검출
  • 일일 정산 검증 배치에서 금액 합산 불일치 감지
  • SP/판매자로부터 이중 입금 신고 접수

대응 절차:

  1. 즉시 해당 건 정산 보류 처리
  2. 중복 정산 금액 확인 및 기록
  3. Stripe에서 중복 Transfer 확인 → Reversal 처리
  4. DB 정산 레코드 정리 (중복 건 status: reversed로 변경)
  5. 원인 분석 후 멱등성(Idempotency) 키 검증 강화

에스컬레이션: 발견 즉시 기술팀 + CEO 보고

오류 유형 4

금액 불일치

원인: 수수료 계산 오류, 환율 변동, 부분 캡처 금액 오류, DB 업데이트 실패

감지 방법:

  • 일일 정산 검증: DB 정산 금액 vs Stripe Transfer 금액 비교
  • 수수료 계산 재검증 (거래 금액 x 수수료율 vs 실제 차감액)
  • SP/판매자 정산 명세서 이의 제기

대응 절차:

  1. 불일치 건 정산 보류
  2. Stripe 원본 데이터와 DB 데이터 대조
  3. 수수료 재계산 및 차액 확인
  4. 차액 발생 시: 추가 송금 또는 다음 정산에서 조정
  5. 수수료 계산 로직 검증 및 수정

에스컬레이션: 차액 $100 초과 시 CEO 보고

오류 유형 5

환불 실패

원인: 잔액 부족 (플랫폼 Stripe 잔액), 원본 결제 분쟁 중, 환불 기한 초과 (180일)

감지 방법:

  • 웹훅 이벤트 refund.failed 수신
  • 환불 큐에서 failed 상태 모니터링
  • 바이어 환불 미수령 신고

대응 절차:

  1. 실패 원인 확인 (Stripe Dashboard > Refunds)
  2. 잔액 부족 시: 플랫폼 Stripe 잔액 충전 후 재시도
  3. 분쟁 중인 경우: 분쟁 해결 후 환불 처리
  4. 기한 초과 시: 계좌 이체를 통한 수동 환불 처리

에스컬레이션: 48시간 내 미해결 시 CEO 보고

3. Stripe 웹훅 재시도 로직

재시도 정책: 최대 3회, 지수 백오프(Exponential Backoff)

시도 대기 시간 실패 시 동작
1차 재시도 1분 후 자동 재시도
2차 재시도 5분 후 자동 재시도 + 로그 기록
3차 재시도 30분 후 자동 재시도 + 알림 발송
최종 실패 - 수동 처리 큐 등록 + 관리자 알림

멱등성 보장: 모든 웹훅 처리는 idempotency_key를 사용하여 중복 처리를 방지합니다.

6. 월별 정산 검증 체크리스트

매월 5일까지 수행 (전월 거래분)

순서 검증 항목 확인 방법
1 총 거래 건수 대조 DB 거래 테이블 vs Stripe Payments 건수
2 총 캡처 금액 대조 DB 캡처 합계 vs Stripe Captured 합계
3 총 정산 금액 대조 DB 정산 합계 vs Stripe Transfers 합계
4 수수료 총액 검증 거래별 수수료 재계산 vs 실제 차감액
5 환불 건수 및 금액 대조 DB 환불 내역 vs Stripe Refunds
6 미처리 건 확인 캡처 실패, 정산 보류, 분쟁 진행 중 건 목록
7 SP 정산 명세서 발행 확인 전체 SP 대상 명세서 생성 여부
8 이상 거래 패턴 점검 비정상 금액, 반복 실패, 이중 처리 여부

절대 금지 사항

  • 검증 없이 수동 정산 처리 금지
  • 오류 건 무시 또는 삭제 금지 — 반드시 원인 분석 후 처리
  • Stripe 대시보드에서 직접 Transfer 생성 금지 (DB 불일치 유발)
  • 프로덕션 DB 직접 수정 금지 — 어드민 대시보드 또는 마이그레이션 사용
