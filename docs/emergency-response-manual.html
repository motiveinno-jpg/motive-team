시나리오 1 — 서비스 장애
Supabase / Stripe / Cloudflare 다운 또는 성능 저하

감지 방법

  • Cloudflare Analytics 이상 트래픽/에러율 급증
  • Supabase Dashboard 상태 이상 감지
  • 사용자 문의 3건 이상 동시 접수
  • 상태 페이지 모니터링: status.supabase.com, status.stripe.com, cloudflarestatus.com

초동대응 (15분 이내)

1
장애 범위 파악: 어떤 서비스가 영향받는지, 전체 장애인지 부분 장애인지 확인
2
외부 서비스 상태 확인: 각 서비스의 Status Page 확인 (자체 문제 vs 외부 장애)
3
CEO 보고: 텔레그램 봇으로 장애 상황 즉시 보고
4
사용자 공지: 플랫폼 내 배너 또는 이메일로 장애 안내 발송

확대 보고

경과 시간조치
15분CEO + 기술 리드에게 초기 상황 보고
30분영향받는 활성 딜 목록 파악, 긴급 딜 수동 처리 검토
1시간미복구 시 대체 프로세스 가동 (수동 결제, 이메일 기반 소통)
4시간미복구 시 전체 사용자 공지, 딜 기한 자동 연장

복구 절차

  • 서비스 복구 확인 후 전체 기능 테스트 (결제, 채팅, 서류 업로드)
  • 장애 중 실패한 트랜잭션 확인 및 수동 재처리
  • 사용자에게 복구 완료 공지

사후 관리

  • 장애 보고서 작성 (원인, 영향, 대응 타임라인, 재발 방지책)
  • 영향받은 딜의 기한 자동 연장 처리
  • 모니터링 강화 (1주간)
시나리오 2 — 결제 사고
이중결제, 무단결제, 정산 오류

감지 방법

  • Stripe Dashboard 이상 거래 감지 (중복 charge, 비정상 금액)
  • 사용자 결제 이의 접수
  • 정산 금액 불일치 자동 알림

초동대응 (15분 이내)

1
즉시 동결: 해당 결제건 Stripe에서 Hold 처리, 추가 캡처 차단
2
원인 파악: Stripe 로그, 서버 로그, 사용자 행동 기록 확인
3
피해자 연락: 영향받은 사용자에게 즉시 연락, 상황 안내
4
CEO 보고: 결제 사고 내역 및 예상 피해 규모 보고

사고 유형별 처리

유형처리 방법소요시간
이중결제중복분 즉시 환불 (Stripe Refund), 사과 메일 발송1시간 이내
무단결제결제 동결, 본인 확인, 미확인 시 전액 환불 + 카드사 신고 안내24시간 이내
정산 오류정산 내역 재계산, 차액 추가 입금 또는 차기 정산 조정3영업일 이내
에스크로 오류캡처 내역 확인, 수동 조정, 양측 확인 후 재정산5영업일 이내

사후 관리

  • 결제 사고 보고서 작성
  • 재발 방지를 위한 결제 로직 점검 및 테스트 강화
  • 영향받은 사용자에게 사후 안내 메일 발송
시나리오 3 — 데이터 유출
개인정보 노출, 무단 접근, DB 유출

감지 방법

  • Supabase 비정상 쿼리 패턴 감지
  • 비인가 IP에서의 API 접근 로그
  • 외부 제보 (사용자, 보안 연구자)
  • 다크웹 모니터링 (향후 구축)

초동대응 (15분 이내)

1
접근 차단: 유출 경로 즉시 차단 (API키 폐기, RLS 점검, 비밀번호 강제 리셋)
2
범위 파악: 유출된 데이터 종류, 영향받은 사용자 수 확인
3
증거 보전: 로그, 접근 기록, 스크린샷 확보 (삭제 방지)
4
CEO 보고: 유출 규모 및 심각도 즉시 보고

사후 관리

  • 유출 사고 보고서 작성 및 공개
  • 보안 정책 강화 (2FA 의무화, IP 화이트리스트 등)
  • 정기 보안 점검 스케줄 수립
시나리오 4 — SP 사고
SP 잠적, 서류 위조, 수출신고 오류

감지 방법

  • SP 48시간 이상 무응답 (시스템 자동 감지)
  • 서류 검증 시 불일치 발견 (관세청 반려, 내부 검증)
  • 제조사 긴급 불만 접수
  • 세관에서 수출신고 반려 통보

초동대응 (15분 이내)

1
즉시 자격 정지: 해당 SP의 모든 신규 딜 배정 중단, 기존 딜 동결
2
제조사 보호: 영향받는 제조사에게 상황 안내, 딜 보호 조치 설명
3
대체 SP 배정: 긴급 딜에 대해 차순위 SP 즉시 배정
4
CEO 보고: 사고 내역 및 영향 딜 수 보고

사고 유형별 처리

유형처리 방법추가 조치
SP 잠적즉시 자격정지, 대체 SP 배정, 미완료 서류 인수인계3회 연락 시도 후 영구 정지
서류 위조즉시 자격정지, 관련 딜 전수 점검, 위조 서류 회수법적 조치 검토, 관세청 신고
수출신고 오류오류 내용 파악, 수정 신고, 대체 관세사 검토페널티 부과, 재교육 의무화

사후 관리

  • SP 사고 이력 기록 (블랙리스트 관리)
  • 영향받은 딜의 일정 재조정 및 제조사 보상 검토
  • SP 심사 기준 강화 검토
시나리오 5 — 법적 분쟁
거래 관련 소송, 계약 분쟁, 지적재산권 문제

감지 방법

  • 법원 소장/내용증명 수신
  • 사용자 법적 조치 예고 메시지
  • 지적재산권 침해 통보

초동대응 (15분 이내)

1
CEO 즉시 보고: 소장/통보 내용 전달
2
법무법인 연락: 소장/통보 원본 전달, 초기 자문 요청
3
증거 보전: 관련 딜 데이터, 채팅 기록, 계약서, 결제 내역 백업 (삭제/수정 금지)
4
내부 소통 제한: 분쟁 관련 외부 발언 금지, 법무법인 지시에 따라 대응

대응 프로세스

단계조치담당
1. 접수소장/통보 접수, CEO 보고관리팀
2. 법률 자문법무법인 초기 검토 및 대응 전략 수립법무법인
3. 증거 수집관련 데이터 보전, 타임라인 정리기술 리드
4. 답변서 작성법무법인 주도 답변서 준비법무법인 + CEO
5. 소송/조정법적 절차 진행법무법인
6. 종결결과 기록, 재발 방지 대책CEO
증거 보전 주의사항: 분쟁 인지 시점부터 관련 데이터(DB, 로그, 채팅, 이메일)의 삭제/수정을 절대 금지합니다. 자동 삭제 정책도 해당 데이터에 대해 일시 중지해야 합니다.

긴급 대응 공통 원칙

  1. 15분 규칙: 모든 긴급 상황은 인지 후 15분 이내 초동대응을 개시합니다.
  2. 보고 우선: 완벽한 파악보다 빠른 초기 보고가 우선입니다. "모르는 것"도 보고합니다.
  3. 증거 보전: 대응 과정에서 로그, 스크린샷, 타임라인을 반드시 기록합니다.
  4. 사용자 우선: 사용자의 자산과 데이터 보호가 최우선 원칙입니다.
  5. 사후 분석: 모든 긴급 상황 종료 후 3영업일 이내 사후 보고서를 작성합니다.
