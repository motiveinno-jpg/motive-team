### Supabase RPC Function: `claim_due_email_jobs`

```sql
CREATE OR REPLACE FUNCTION claim_due_email_jobs(
  p_batch_size INTEGER DEFAULT 10,
  p_worker_id TEXT DEFAULT 'n8n-w3'
)
RETURNS SETOF email_queue
LANGUAGE plpgsql
AS $$
BEGIN
  RETURN QUERY
  UPDATE email_queue
  SET
    status = 'claimed',
    claimed_by = p_worker_id,
    claimed_at = NOW()
  WHERE id IN (
    SELECT id
    FROM email_queue
    WHERE status = 'pending'
      AND send_at <= NOW()
    ORDER BY send_at ASC
    LIMIT p_batch_size
    FOR UPDATE SKIP LOCKED
  )
  RETURNING *;
END;
$$;
```

## W1: Content Decompose Engine

**Workflow name:** `W1_Content_Decompose_On_Blog_Published`
**Purpose:** When a blog post is deployed, decompose it into channel-specific variants (LinkedIn, Twitter/X, Instagram, email newsletter) via an AI Edge Function.

### Node Chain

```
[Webhook Trigger] -> [Validate Secret] -> [Call Edge Function] -> [Flatten Variants] -> [Save to DB] -> [Telegram Notify]
```

### Node 1: Webhook Trigger

- **Type:** Webhook
- **Method:** POST
- **Path:** `/w1-content-decompose`
- **Authentication:** Header Auth
- **Response Mode:** Last Node

### Node 2: Validate Secret (Code)

```javascript
const webhookSecret = $env.WHISTLE_WEBHOOK_SECRET;
const receivedSecret = $input.first().headers['x-webhook-secret'];

if (receivedSecret !== webhookSecret) {
  throw new Error('Invalid webhook secret');
}

const body = $input.first().json;

if (!body.post_id || !body.content || !body.title) {
  throw new Error('Missing required fields: post_id, content, title');
}

return [{
  json: {
    post_id: body.post_id,
    title: body.title,
    content: body.content,
    language: body.language || 'ko',
    channels: body.channels || ['linkedin', 'twitter', 'instagram', 'email'],
  },
}];
```

### Node 3: Call Edge Function (HTTP Request)

- **Type:** HTTP Request
- **Method:** POST
- **URL:** `{{ $env.SUPABASE_FUNCTIONS_BASE_URL }}/decompose-content`
- **Headers:**
  - `Authorization`: `Bearer {{ $env.SUPABASE_SERVICE_ROLE_KEY }}`
  - `Content-Type`: `application/json`
- **Body:**

```json
{
  "post_id": "{{ $json.post_id }}",
  "title": "{{ $json.title }}",
  "content": "{{ $json.content }}",
  "language": "{{ $json.language }}",
  "channels": "{{ $json.channels }}"
}
```

- **Timeout:** 60000ms (AI generation may take time)

### Node 4: Flatten Variants (Code)

```javascript
const variants = $input.first().json.variants;
const postId = $('Validate Secret').first().json.post_id;

if (!variants || !Array.isArray(variants)) {
  throw new Error('No variants returned from Edge Function');
}

return variants.map((v) => ({
  json: {
    post_id: postId,
    channel: v.channel,
    language: v.language || 'ko',
    variant_no: v.variant_no || 1,
    title: v.title,
    body: v.body,
    hashtags: v.hashtags || [],
    media_urls: v.media_urls || [],
    status: 'draft',
  },
}));
```

### Node 5: Save to DB (Supabase)

- **Type:** Supabase
- **Operation:** Insert
- **Table:** `content_variants`
- **On Conflict:** `post_id, channel, language, variant_no` -> Update
- **Fields:** All fields from the flattened variant

### Node 6: Telegram Notify

- **Type:** Telegram
- **Operation:** Send Message
- **Chat ID:** `{{ $env.TG_CEO_CHAT_ID }}`
- **Message:**

```
[Whistle Content Engine]
Blog decomposed: {{ $('Validate Secret').first().json.title }}
Variants: {{ $input.all().length }} created
Channels: {{ [...new Set($input.all().map(i => i.json.channel))].join(', ') }}
```

## W2: Lead Capture Engine

**Workflow name:** `W2_Lead_Capture_And_Welcome`
**Purpose:** Capture leads from the website form, validate, segment, store in Supabase, send a welcome email, and queue the nurture sequence.

### Node Chain

```
[Webhook Trigger] -> [Spam Check] -> [Normalize] -> [Derive Segment] -> [Check Existing] -> [Upsert Lead] -> [Send Welcome] -> [Queue Next Email] -> [Telegram Notify]
```

### Node 1: Webhook Trigger

- **Type:** Webhook
- **Method:** POST
- **Path:** `/w2-lead-capture`
- **Response Mode:** Last Node

### Node 2: Spam/Consent Check (Code)

```javascript
const body = $input.first().json;

// Honeypot check
if (body.website || body.hp_field) {
  throw new Error('Spam detected: honeypot field filled');
}

// Email required
if (!body.email || !body.email.includes('@')) {
  throw new Error('Invalid or missing email');
}

// Consent required
if (!body.consent) {
  throw new Error('Marketing consent not given');
}

return [{
  json: {
    email: body.email,
    name: body.name || null,
    company: body.company || null,
    phone: body.phone || null,
    source: body.source || 'website',
    consent: true,
    raw: body,
  },
}];
```

### Node 3: Normalize Lead Data (Code)

```javascript
const data = $input.first().json;

return [{
  json: {
    email: data.email.toLowerCase().trim(),
    name: data.name ? data.name.trim() : null,
    company: data.company ? data.company.trim() : null,
    phone: data.phone ? data.phone.replace(/[^0-9+\-]/g, '') : null,
    source: data.source,
    consent_at: new Date().toISOString(),
  },
}];
```

### Node 4: Derive Segment (Code)

```javascript
const data = $input.first().json;
const email = data.email;
const source = data.source;

let segment = 'global_buyer';

// Korean SME detection
const isKoreanDomain = email.endsWith('.kr') ||
  email.endsWith('@naver.com') ||
  email.endsWith('@daum.net') ||
  email.endsWith('@hanmail.net') ||
  email.endsWith('@kakao.com');

if (isKoreanDomain) {
  segment = 'kr_sme';
}

// Alibaba source detection
if (source === 'alibaba' || source === 'alibaba_landing') {
  segment = 'alibaba_brand';
}

// Override from explicit source param
if (source === 'buyer_landing') {
  segment = 'global_buyer';
}

return [{
  json: {
    ...data,
    segment,
  },
}];
```

### Node 5: Check Existing Lead (Supabase)

- **Type:** Supabase
- **Operation:** Select
- **Table:** `leads`
- **Filter:** `email` eq `{{ $json.email }}`
- **Limit:** 1

### Node 6: Upsert Lead (Code + Supabase)

**IF existing lead found (branch):**

- **Type:** Supabase
- **Operation:** Update
- **Table:** `leads`
- **Filter:** `email` eq `{{ $json.email }}`

**IF no existing lead (branch):**

## W3: Email Sequence Dispatcher

**Workflow name:** `W3_Email_Sequence_Dispatcher`
**Purpose:** Every hour, claim pending email jobs, render templates, send via Resend, and schedule the next step.

- **Type:** Schedule Trigger

### Node 6: Render Variables (Code)

```javascript
const template = $input.first().json;
const lead = $('Check Lead Active').first().json;

if (!template || !template.subject) {
  throw new Error(`Template not found for key=${$json.template_key}, step=${$json.step_no}`);
}

let subject = template.subject;
let htmlBody = template.html_body;
let textBody = template.text_body || '';

const variables = {
  '{{name}}': lead.name || 'there',
  '{{company}}': lead.company || 'your company',
  '{{email}}': lead.email,
  '{{app_url}}': $env.APP_BASE_URL,
  '{{unsubscribe_url}}': `${$env.APP_BASE_URL}/unsubscribe?email=${encodeURIComponent(lead.email)}`,

for (const [placeholder, value] of Object.entries(variables)) {
  subject = subject.replaceAll(placeholder, value);
  htmlBody = htmlBody.replaceAll(placeholder, value);
  textBody = textBody.replaceAll(placeholder, value);
}

return [{
  json: {
    to: lead.email,
    subject,
    html: htmlBody,
    text: textBody,
    lead_id: lead.id,
    step_no: $json.step_no,
    segment: $json.segment,
    template_key: $json.template_key,
```

## W4: Lead Scoring Engine

**Workflow name:** `W4_Resend_Webhook_Lead_Scoring`
**Purpose:** Receive Resend email events, deduplicate, apply scoring rules, and tag high-score leads as call candidates.

### Node Chain

```
[Webhook Trigger] -> [Verify Signature] -> [Deduplicate] -> [Apply Score] -> [Update Lead] -> [Check Threshold] -> [Telegram Alert]
```

### Node 1: Webhook Trigger

- **Type:** Webhook
- **Method:** POST
- **Path:** `/w4-resend-events`
- **Response Mode:** Immediately (200 OK)

### Node 2: Verify Signature (HTTP Request to Edge Function)

- **Type:** HTTP Request
- **Method:** POST
- **URL:** `{{ $env.SUPABASE_FUNCTIONS_BASE_URL }}/verify-resend-webhook`
- **Headers:**
  - `Authorization`: `Bearer {{ $env.SUPABASE_SERVICE_ROLE_KEY }}`
  - `Content-Type`: `application/json`
- **Body:**

```json
{
  "payload": "{{ JSON.stringify($input.first().json) }}",
  "signature": "{{ $input.first().headers['svix-signature'] }}",
  "timestamp": "{{ $input.first().headers['svix-timestamp'] }}",
  "svix_id": "{{ $input.first().headers['svix-id'] }}"
}
```

### Node 3: Deduplicate (Code)

```javascript
const svixId = $input.first().headers['svix-id'];
const eventType = $input.first().json.type;
const data = $input.first().json.data;

// Check if svix_id already exists in email_events
// This is done via a Supabase select before this node in practice
// If found, return empty to skip processing

return [{
  json: {
    svix_id: svixId,
    event_type: eventType,
    resend_email_id: data.email_id,
    to_email: data.to?.[0] || data.email,
    payload: $input.first().json,
  },
}];
```

**Supabase check (before Code node):**

- **Type:** HTTP Request
- **Method:** GET
- **URL:** `{{ $env.SUPABASE_URL }}/rest/v1/email_events?svix_id=eq.{{ headers['svix-id'] }}&select=id&limit=1`
- **IF row exists -> stop processing**

### Node 4: Apply Score (Code)

```javascript
const SCORING_RULES = {
  'email.opened': 5,
  'email.clicked': 15,
  'email.bounced': -20,
  'email.complained': -30,
  'email.delivered': 1,
};

const eventType = $json.event_type;
const scoreDelta = SCORING_RULES[eventType] || 0;

return [{
  json: {
    ...$json,
    score_delta: scoreDelta,
  },
}];
```

### Node 5: Save Event + Update Lead Score

**5a: Save event to `email_events`:**

- **Type:** Supabase Insert
- **Table:** `email_events`
- **Fields:** `svix_id`, `event_type`, `resend_email_id`, `lead_id` (resolved via email lookup), `payload`

**5b: Look up lead by email and get current score:**

- **Type:** HTTP Request
- **Method:** GET
- **URL:** `{{ $env.SUPABASE_URL }}/rest/v1/leads?email=eq.{{ $json.to_email }}&select=id,score,call_candidate`

**5c: Update lead score:**

- **Type:** HTTP Request
- **Method:** PATCH
- **URL:** `{{ $env.SUPABASE_URL }}/rest/v1/leads?id=eq.{{ lead.id }}`
- **Body:**

```json
{
  "score": "{{ lead.score + $json.score_delta }}",
  "updated_at": "{{ DateTime.now().toISO() }}"
}
```

### Node 6: Check Threshold (Code)

```javascript
const CALL_THRESHOLD = 25;
const lead = $('Look up lead').first().json;
const newScore = lead.score + $json.score_delta;
const wasCallCandidate = lead.call_candidate;
const isNowCallCandidate = newScore >= CALL_THRESHOLD;

if (isNowCallCandidate && !wasCallCandidate) {
  return [{
    json: {
      lead_id: lead.id,
      email: $json.to_email,
      new_score: newScore,
      is_new_call_candidate: true,
    },
  }];
}

return [];
```

**IF `is_new_call_candidate` is true:**

Update lead `call_candidate = true`:

- **Type:** HTTP Request
- **Method:** PATCH
- **URL:** `{{ $env.SUPABASE_URL }}/rest/v1/leads?id=eq.{{ $json.lead_id }}`
- **Body:**

```json
{
  "call_candidate": true,
  "tags": "{{ JSON.stringify([...existingTags, 'call_candidate']) }}",
  "updated_at": "{{ DateTime.now().toISO() }}"
}
```

### Node 7: Telegram Alert

- **Type:** Telegram
- **Operation:** Send Message
- **Chat ID:** `{{ $env.TG_CEO_CHAT_ID }}`
- **Message:**


## W5: Daily Report

**Workflow name:** `W5_Daily_Report`
**Purpose:** Every day at 18:00 KST, compile daily metrics and send a summary to the CEO via Telegram.

### Node Chain

```
[Cron Trigger] -> [Query Metrics] -> [Format Report] -> [Save Metrics] -> [Send Telegram]
```

### Node 1: Cron Trigger

- **Type:** Schedule Trigger
- **Cron Expression:** `0 9 * * *` (09:00 UTC = 18:00 KST)

### Node 2: Query Metrics (Code + HTTP Requests)

Run these queries in parallel against Supabase:

**2a: New leads today**

```
GET {{ $env.SUPABASE_URL }}/rest/v1/leads?created_at=gte.{{ DateTime.now().startOf('day').toISO() }}&select=id&limit=1000
```
-> Count results

**2b: Emails sent today**

```
GET {{ $env.SUPABASE_URL }}/rest/v1/email_queue?status=eq.sent&sent_at=gte.{{ DateTime.now().startOf('day').toISO() }}&select=id&limit=1000
```
-> Count results

**2c: Opens today**

```
GET {{ $env.SUPABASE_URL }}/rest/v1/email_events?event_type=eq.email.opened&processed_at=gte.{{ DateTime.now().startOf('day').toISO() }}&select=id&limit=1000
```
-> Count results

**2d: Clicks today**

```
GET {{ $env.SUPABASE_URL }}/rest/v1/email_events?event_type=eq.email.clicked&processed_at=gte.{{ DateTime.now().startOf('day').toISO() }}&select=id&limit=1000
```
-> Count results

**2e: Call candidates total**

```
GET {{ $env.SUPABASE_URL }}/rest/v1/leads?call_candidate=eq.true&select=id&limit=1000
```
-> Count results

**2f: Total leads**

```
GET {{ $env.SUPABASE_URL }}/rest/v1/leads?select=id&limit=10000
```
-> Count results

### Node 3: Format Report (Code)

```javascript
const today = DateTime.now().setZone('Asia/Seoul').toFormat('yyyy-MM-dd');
const newLeads = $('New leads today').first().json.length || 0;
const emailsSent = $('Emails sent today').first().json.length || 0;
const opens = $('Opens today').first().json.length || 0;
const clicks = $('Clicks today').first().json.length || 0;
const callCandidates = $('Call candidates total').first().json.length || 0;
const totalLeads = $('Total leads').first().json.length || 0;

const openRate = emailsSent > 0 ? ((opens / emailsSent) * 100).toFixed(1) : '0.0';
const clickRate = emailsSent > 0 ? ((clicks / emailsSent) * 100).toFixed(1) : '0.0';

const report = `[Whistle Daily Report] ${today}

New Leads: ${newLeads}
Total Leads: ${totalLeads}
Emails Sent: ${emailsSent}
Opens: ${opens} (${openRate}%)
Clicks: ${clicks} (${clickRate}%)
Call Candidates: ${callCandidates}

#whistle #daily`;

return [{
  json: {
    report,
    metrics: {
      date: today,
      new_leads: newLeads,
      emails_sent: emailsSent,
      opens,
      clicks,
      bounces: 0,
      call_candidates: callCandidates,
      total_leads: totalLeads,
    },
  },
}];
```

### Node 4: Save Metrics (Supabase)

- **Type:** Supabase
- **Operation:** Upsert
- **Table:** `daily_metrics`
- **Conflict Column:** `date`
- **Fields:** All fields from `metrics` object

### Node 5: Send Telegram

- **Type:** Telegram
- **Operation:** Send Message
- **Chat ID:** `{{ $env.TG_CEO_CHAT_ID }}`
- **Message:** `{{ $json.report }}`
- **Parse Mode:** None (plain text)

## Implementation Notes

### Deployment Order

1. Create all Supabase tables and the RPC function first.
2. Seed `email_templates` with initial nurture sequences for each segment.
3. Deploy Edge Functions: `decompose-content`, `verify-resend-webhook`.
4. Import workflows into n8n on PC4 (Ops).
5. Configure all environment variables in n8n credentials/variables.
6. Activate workflows in order: W5 -> W4 -> W3 -> W2 -> W1.
7. Test each webhook endpoint with curl before connecting to production.

### Error Handling

- All HTTP Request nodes should have retry enabled (max 3 retries, 5s backoff).
- Failed email sends should update `email_queue.status` to `'failed'` with `error_message`.
- Telegram notifications should be wrapped in try/catch -- notification failure must not break the main flow.

### Monitoring

- n8n execution history provides built-in monitoring.
- W5 (Daily Report) serves as the primary health check.
- If W5 stops sending, investigate n8n container health on PC4.

### Resend Webhook Setup

Register the webhook URL in Resend dashboard:
- **URL:** `https://<n8n-domain>/webhook/w4-resend-events`
- **Events:** `email.opened`, `email.clicked`, `email.bounced`, `email.delivered`, `email.complained`
