## 1. 핵심 컨셉

**"한국에서 지금 유행하는 진짜 제품을 찾고, 검증하고, 연결한다"**

Whistle AI는 양면 마켓플레이스(Two-Sided Marketplace)입니다.
- **바이어**: 한국 트렌드 제품 검색 → 원산지 검증 → 제조사 연결
- **셀러**: 상품 등록 → AI 수출 분석 → 바이어 매칭
- **Whistle Admin**: 양쪽이 못하는 것을 대행 (공장 컨택, 번역, 물류 등)

```
┌─────────────────────────────────────────────────────────────────┐
│                    Whistle AI Platform                          │
│                                                                 │

│  │  Buyer   │────▶│  Whistle DB  │◀────│  Seller  │            │
│  │  Portal  │     │  (Core Hub)  │     │  Portal  │            │
│  └────┬─────┘     └──────┬───────┘     └────┬─────┘            │
│       │                  │                   │                  │
```

### 4-2. 셀러-바이어 싱크 매핑

```
┌──────────────────────────────────────────────────────────────┐
│                     매칭 & 싱크 엔진                          │
│                                                              │
│  바이어 검색 "달팽이크림"                                      │
│       │                                                      │
│       ├──▶ [네이버 API] → 내수 상품 결과                       │
│       │         │                                            │
│       │         ▼                                            │
│       │    products 테이블에 캐싱                               │
│       │    (source: 'naver_api')                              │
│       │         │                                            │
│       │         ▼                                            │
│       ├──▶ [Whistle DB] → 셀러 등록 상품 매칭                  │
│       │    (source: 'seller_direct')                          │
│       │         │                                            │
│       │         ▼                                            │
│       │    ★ 동일 상품 감지 (제조사+브랜드 매칭)                  │
│       │    → naver 상품에 seller_id 연결                       │
│       │    → 바이어에게 "검증된 공급자 있음" 표시                  │
│       │                                                      │
│       ▼                                                      │
│  바이어가 인콰이어리 제출                                       │
│       │                                                      │
│       ├──▶ 셀러 등록 상품인 경우                                │
│       │    → 셀러에게 직접 알림                                 │
│       │    → 셀러가 직접 대응 OR Whistle 대행 선택               │
│       │                                                      │
│       ├──▶ 네이버 상품인 경우 (셀러 미등록)                      │
│       │    → Whistle Admin이 제조사 추적                       │
│       │    → 제조사 확인 후 셀러 가입 유도                       │
│       │    → 매칭 완료 후 양쪽 연결                             │
│       │                                                      │
│       └──▶ 맞춤 소싱인 경우                                    │
│            → Whistle Admin 전담 처리                          │
│            → 2~3개 공급사 옵션 제안                             │
└──────────────────────────────────────────────────────────────┘
```

### 4-3. 핵심 싱크 규칙

1. **네이버 상품 → 셀러 매핑**: 네이버에서 검색된 상품의 `maker`가 셀러 DB의 `company_name`과 일치하면 자동 매핑
2. **셀러 상품 → 네이버 매핑**: 셀러가 등록한 상품의 네이버 쇼핑 URL이 있으면 `naver_product_id`로 연결
3. **인콰이어리 라우팅**: 매칭된 셀러가 있으면 셀러에게 먼저, 없으면 Admin으로 자동 라우팅
4. **검색 결과 통합**: 바이어 검색 시 "Whistle 검증" (셀러 등록) + "Korea Market" (네이버) 결과를 통합 표시
